## Пространства имен

> **Пространство имен** — это группа логически связанных имен сложных типов (классов, структур, интерфейсов) и подчиненных пространств имен

Назначения пространств имен:

- Логическая группировка и структурирование кода
- Разрешение конфликтов имен

- На данный момент можно рассматривать классы, структуры и интерфейсы как контейнеры с переменными и методами. Далее эти объекты будут подробно раскрыты

Пространство имён создаёт область видимости, в которой имена внутри него уникальны и видимы, а за пределами пространства имён для доступа к этим именам нужно либо использовать квалифицированное имя с указанием пространства имён, либо директиву `using`

### Объявление пространства имен

Пространство имен может быть блочным с явно указанными границами в файле с кодом:

```cs
namespace Addons {
    
    namespace MyAddons {
        
        public class Picture;
        
        public class Video;
    }
    
    namespace TheirAddons {
        
        public class Picture;
        
        public class Video;
    }
}
```

Объявление на уровне файла (начиная с .NET 6, C\# 10):

```cs
namespace MyNamespace;
class MyClass1;
class MyClass2;
```

Пример использования

```cs
Addons.MyAddons.Picture.Show();
Addons.TheirAddons.Picture.Hide();

MyNamespace.MyClass1.Method();
MyNamespace.MyClass1.Method();

// -- или --

using MyNamespace;
MyClass1.Method();
MyClass1.Method();
```

Начиная с версии .NET 6, C\# 10 появилась возможность объявлять директиву `using` глобально для проекта (`global using`) и подключать частоиспользуемые пространства имен неявно (`implicit using`)

## Области видимости

> **Область видимости** (scope) — это часть программы, в пределах которой доступна и видима определённая переменная или идентификатор. Вне этой области переменная перестаёт существовать или становится недоступной.

Существует несколько уровней областей видимости:

- ~~Уровень сборки~~ (проекта). В C\# **не выделяется** уровень области под всю сборку
- **Уровень пространства имен**. Определения типов видимы в рамках данного пространства имен и везде, где это пространство имен подключается. Переменные на уровне пространства имен определяться не могут
- **Уровень типа**. Переменные видимы на уровне всей структуры (`int`, `ConsoleKeyInfo`, ...) или класса (`Console`, `Math`,  `List<T>`, ...)
- **Уровень метода**. Переменные видны только в рамках текущего метода и внутри вложенных подметодов ([[03 Локальные функции|локальных функций]]\*)
- **Уровень блока**. В методе может быть определено множество вложенных и невложенных блоков кода. Каждый новый вложенный блок кода представляет более и более глубокий уровень области видимости

Блоком кода могут выступать следующие конструкции:

- Оператор `if-else`, `switch-case`
- Оператор `while`, `do-while`
- Оператор `for`, `foreach`
- Оператор блока `{ }`

> **Оператор блока** существует только для того, чтобы явно разграничить область видимости переменных. Оператор блока ничего не возвращает

```csharp
{
    int jar = 308;
}

jar++; //не существует в текущем контексте
```
