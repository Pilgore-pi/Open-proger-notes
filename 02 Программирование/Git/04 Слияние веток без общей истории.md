
Ранее, мы объединили 2 репозитория в один общий репозиторий, но в разные ветки. Таким образом, мы получили 2 чужеродные ветки без общей истории. При слиянии таких веток может произойти полная потеря одной из веток, поэтому слияние в таких случаях запрещено по умолчанию. При попытки слияния чужеродных веток возникает ошибка: `fatal: refusing to merge unrelated histories`.

Так как у нас учебный пример, мы отключим этот запрет.

> Внимание! Эта операция может удалить важные данные, желательно копировать репозиторий перед выполнением ненадежных операций

Скопируем репозиторий в локальную папку:

```git
# git clone <URL_source> <URL_copy>
git clone https://github.com/Hero/ProgerNotes "C:/MyNotes/ProgerNotes_copy" 
```
git clone https://github.com/Pilgore-pi/ProgerNotes "C:\Andrey\Конспекты\Главное Хранилище\Backups\ProgerNotes Git" 

В URL файла можно использовать прямые или обратные слеши, если каталога `ProgerNotes_copy` не существует, Git создаст его

На практике, при попытке получения изменений со слиянием, может произойти ошибка: `fatal: refusing to merge unrelated histories` (отказано в слиянии несвязанных историй). Такое обычно происходит, если осуществляется попытка выполнить слияние веток, у которых нет общего родителя (грубо говоря, слияние двух веток с разных деревьев). По умолчанию такое действие запрещено, чтобы избежать фатальных ошибок невнимательного программиста. Так как у нас учебный пример, то можно отключить этот запрет
