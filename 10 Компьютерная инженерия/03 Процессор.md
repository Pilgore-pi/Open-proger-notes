**CPU (Central Processing Unit), ЦПУ (ЦП)** — Центральное процессорное устройство, интегральная схема, исполняющая машинные инструкции. Он состоит из миллиардов транзисторов, сопоставимых с молекулами ДНК. Множество транзисторов объединяются в логические вентили, которые объединяются в более сложные логические механизмы, позволяющие складывать, умножать, сравнивать числа. Данные для обработки берутся из оперативной памяти. Оперативная память значительно медленней процессора, поэтому, для сокращения простоя, используется кэш-память трех (иногда четырех) уровней: L1, L2, L3. От L1 к L3 увеличивается объем памяти и уменьшается скорость. Чем больше кэш, тем меньше процессор обращается к оперативной памяти. Скорость кэш-памяти около *120 Гб/с*

**Процессор (CPU, Central Processing Unit, ЦПУ)** — центральное процессорное устройство, интегральная схема, исполняющая машинные инструкции программ. Процессор состоит из миллиардов транзисторов, сопоставимых с молекулами ДНК, которые объединяются в логические вентили, позволяющие выполнять арифметические операции (сложение, умножение), сравнение чисел и логические операции. Данные для обработки берутся из оперативной памяти, однако из-за её относительной медлительности используется кэш-память для ускорения доступа к часто используемым данным.

## Тактовая частота

?Что такое частота?

Частота процессора меняется путем понижения напряжения в зависимости от нагрузки для экономии энергии. Частота определяется 3 факторами:

- Базовая частота
- Множители. для каждого ядра процессор использует 1 множитель, который увеличивает частоту
- Напряжение ядра. Повышенное напряжение необходимо для получения стабильно высокой тактовой частоты. При повышении напряжения увеличивается тепловыделение.

Тактовая частота процессора рассчитывается по формуле:

`Тактовая_частота = Базовая_частота × множитель`

## Многопоточность

**Регистр** — сверхбыстрая оперативная память (СОЗУ) внутри процессора, предназначенная прежде всего для хранения промежуточных результатов вычисления.

Регистр представляет собой цифровую электронную схему, служащую для временного хранения двоичных чисел. В процессоре имеется значительное количество регистров, большая часть которых используется самим процессором и недоступна программисту. Имеются также регистры, которые в принципе программно доступны, но обращение к ним осуществляется из программ операционной системы (например, управляющие регистры и теневые регистры дескрипторов сегментов). Этими регистрами пользуются в основном разработчики операционных систем.

Доступ к значениям, хранящимся в регистрах, как правило, в несколько раз быстрее, чем доступ к ячейкам оперативной памяти (даже если кэш-память содержит нужные данные), но объём оперативной памяти намного превосходит суммарный объём регистров (объём среднего модуля оперативной памяти сегодня составляет 4 Гб, суммарная ёмкость регистров общего назначения/данных для процессора Intel 80x86 16 бит * 4 = 64 бита (8 байт)).

Больше информации о регистрах – [https://sites.google.com/site/tehnorainfo/sistemnoe-programmirovanie/lekcia-2-registry-processora-i-ih-naznacenie](https://sites.google.com/site/tehnorainfo/sistemnoe-programmirovanie/lekcia-2-registry-processora-i-ih-naznacenie)

При исполнении программ могут использоваться _прерывания_. Один процесс временно приостанавливается, при этом запускается более приоритетный процесс (обработчик прерываний). По окончании исполнения обработчика прерываний прерванный процесс продолжается. Прерывания может генерировать программист и сам процессор при определенных обстоятельствах – исключениях.

Например:

- Деление на ноль
- Отладочное исключение. Генерируется для отладки
- Немаскируемое прерывание. Генерируется при ошибках ОЗУ и невосстановимых ошибках железа
- Точка останова. Используется для отладки
- Переполнение
- Выход за пределы
- Ошибка защиты

> Также прерывания могут генерироваться подключаемыми устройствами

Hyper-treading (гиперпоточность, hyper-treading technology, HTT или HT) – технология, разработанная компанией Intel, которая позволяет использовать 2 потока в одном ядре. Работа второго потока происходит в тот момент, когда простаивает первый, поэтому эта технология не может полноценно заменить физическое ядро на виртуальное. В компьютере понятия виртуального и физического ядра неразличимы (они называются логическими).

Каждое физическое ядро:

1. может хранить состояние сразу 2 потоков
2. содержит по одному набору регистров и по одному контроллеру прерываний на каждый логический процессор

Производительность HTT процессора варьируется от программы к программе, так как они по-разному используют его ресурсы.
