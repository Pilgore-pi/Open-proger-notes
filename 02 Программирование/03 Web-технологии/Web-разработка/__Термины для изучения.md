> **TCP** — Transmission Control Protocol, основной протокол передачи данных в интернете. Обеспечивает контроль порядка передаваемых данных и отслеживание возникающих ошибок с помощью контрольных сумм.

> **HTTP** — HyperText Transfer Protocol, протокол, являющийся надстройкой над TCP. Данный протокол поддерживает специальные методы запросов GET, POST, DELETE, PUT и так далее

> **Сокет** — это программный интерфейс для обмена данными между процессами, которые могут находиться на одном компьютере или на разных компьютерах в сети. В контексте TCP, сокет — это комбинация IP-адреса и номера порта, которая идентифицирует конкретное соединение.

**DNS (Domain Name System)**: Система, которая преобразует доменные имена в IP-адреса.

**DHCP (Dynamic Host Configuration Protocol)**: Протокол, который автоматически назначает IP-адреса устройствам в сети.

**NAT (Network Address Translation)**: Технология, которая позволяет нескольким устройствам в локальной сети использовать один публичный IP-адрес для доступа в Интернет.

**Firewall (Межсетевой экран)**: Система безопасности, которая контролирует и фильтрует сетевой трафик.

### Порты

> **Сетевой порт** — это виртуальный аналог физического порта. В компьютере есть множество портов (USB, HDMI, SATA), а для IP-адреса по аналогии также есть порты.

> **Виртуальный порт** — это числовой идентификатор (от 1 до 65535 включительно), который используется для адресации конкретных процессов или служб на узле сети. Порты позволяют одному IP-адресу обслуживать несколько сетевых соединений одновременно. Порты делятся на три категории:

- **Well-known ports (0-1023)**: Зарезервированы для системных или широко используемых служб (например, HTTP использует порт 80).
- **Registered ports (1024-49151)**: Зарегистрированы для определенных служб и приложений.
- **Dynamic or private ports (49152-65535)**: Используются для временных или частных соединений.

IP-адрес — это уникальный адрес, который идентифицирует узел в сети. Существует два основных типа IP-адресов:

- **IPv4**: Состоит из четырех байт (например, `192.168.1.1`).
- **IPv6**: Состоит из восьми групп по четыре шестнадцатеричных цифры (например, `2001:0db8:85a3:0000:0000:8a2e:0370:7334`).

**Виды IP-адресов**

- **Частные IP-адреса**: Используются внутри локальных сетей и не маршрутизируются в Интернете (например, `192.168.0.0 - 192.168.255.255`).
- **Публичные IP-адреса**: Маршрутизируются в Интернете и должны быть уникальными в глобальной сети.

### Модель OSI

Это сетевая модель, которая описывает и стандартизирует функции сетевой связи без учета конкретных технологий и протоколов. Данная модель поддерживает множество различных стандартов: HTTP, UDP, FTP, Telnet и другие. Всего таких протоколов более 100.

Состоит из 7 логически независимых уровней, каждый из которых выполняет свою функцию и обладает своим набором протоколов. Взаимодействие между уровнями происходит через четко определенные интерфейсы. Данные, передаваемые по сети, проходят через все уровни модели OSI, начиная с прикладного уровня на стороне отправителя и заканчивая физическим уровнем. На стороне получателя данные проходят в обратном порядке

![[Схема модели OSI.png]]

Самый нижний слой отвечает за физическое представление данных, то есть за то, как данные передаются по проводам или с помощью радиоволн, а самый верхний — за то, как приложения взаимодействуют с сетью.

Нижний слой оперирует такими понятиями, как «тип кабеля» или «тип коннектора», а верхний — такими, как **HTTP** или **API**.

Модель OSI используется как эталон для понимания и разработки сетевых протоколов и технологий. Она помогает стандартизировать сетевые взаимодействия и обеспечивает совместимость между различными устройствами и системами.
