
> **JavaScript** — это мультипарадигменный язык программирования с динамической типизацией, который поддерживает объектно-ориентированный, императивный и функциональный стили программирования. Он является реализацией стандарта ECMAScript и используется в основном для создания интерактивных веб-страниц, но также применяется и вне браузеров, например, на сервере (с помощью **Node.js**)

Автор языка — Брендан Айх, создатель **JavaScript**, разработчик в компании **Netscape**, позже участник создания **Mozilla Foundation**

Язык JavaScript является:

* Интерпретируемым
* Императивным
* Высокоуровневым
* Сценарным
* Мультипарадигменным:
    * Объектно-ориентированным (**прототипным**)
    * Функциональным
    * Событийно-ориентированным
* Динамически типизированным
* Слабо типизированным
* Неявно типизированным
* Языком общего назначения

На [GitHub](https://github.com/tc39/ecma262) размещена спецификация ECMAScript (стандарт JavaScript)

## Платформа ECMAScript

> Язык JavaScript подчиняется стандарту ECMAScript, определяемому организацией TC39

Стандарт ECMAScript определяет единую систему типов и синтаксис языка, позволяя различным реализациям JavaScript работать на разных платформах и окружениях

Существует несколько основных реализаций JavaScript:

- V8 (Google Chrome, Node.js)
- SpiderMonkey (Mozilla Firefox)
- JavaScriptCore (Apple Safari)
- Chakra (Microsoft Edge)

Другие языки могут компилироваться в JavaScript или работать на его основе:

| Язык       | Реализация с JavaScript                                    |
| ---------- | ---------------------------------------------------------- |
| TypeScript | Надмножество JavaScript с поддержкой статической типизации |
| CoffeeScript | Язык, компилирующийся в JavaScript                       |
| Dart       | Язык, компилирующийся в JavaScript                        |
| Elm        | Функциональный язык, компилирующийся в JavaScript         |
| ReScript   | Язык с синтаксисом OCaml, компилирующийся в JavaScript    |
| Flow       | Система типов для JavaScript                              |

## Совместимость стандартов ECMAScript

JavaScript до 2009 года разрабатывался и развивался по принципу полной обратной совместимости, чтобы код был универсальным и работал во всех браузерах. Однако, такой подход привел к накоплению неудачных решений и ошибок в JavaScript, поэтому, с выходом в 2009 году ECMAScript 5 (ES5), JavaScript потерял обратную совместимость, так как были добалвены новые возможности и изменены некоторые из существующих. Чтобы устаревший код работал, как и раньше, по умолчанию новые возможности отключены. Поэтому в современном коде приходится использовать специальный **строгий режим**, который явно указывает на использование современного кода.

Чтобы использовать **строгий режим** в начале файла с кодом пишется директива `"use strict";` или `'use strict';`

```javascript
"use strict";

// этот код работает в современном режиме
...
```

> Директива `"use strict";` должна находится в первой логической строке файла, иначе директива может не сработать. Допускается использование пробельных символов и комментариев перед директивой

Если JavaScript в коде используются новые конструкции, такие как **классы** и **модули**, которых не было в старых версиях, то применять директиву не нужно, так как браузер автоматически определит, что используется современный JS

> Если применен режим `'use strict'`, то отменить его уже нельзя

## TypeScript

> **TypeScript** — это язык программирования, разработанный Microsoft в 2012 году, который расширяет возможности JavaScript за счёт добавления строгой статической типизации. Это надстройка над JavaScript, позволяющая разработчикам указывать типы данных для переменных, функций и объектов ещё на этапе разработки, что помогает выявлять ошибки до выполнения программы и создавать более надёжный и масштабируемый код

Автор языка TypeScript — Андерс Хейлсберг, создатель C\#

При выполнении TypeScript программы компилятор TypeScript **`tsc`** выполняет анализ кода и валидацию типов, затем, после успешного анализа кода, TypeScript ретранслируется в JavaScript. Сгенерированный код JavaScript уже не содержит никаких аннотаций типов и работает как обычный динамически типизируемый JS

- TypeScript автоматически анализирует код и, где нужно, добавляет директиву **`"use strict"`**

TypeScript позволяет использовать абстракции, которых нет в чистом JavaScript:

- Аннотации типов

```typescript
let age: number = 25;

function greet(name: string): void {
    console.log("Hello, " + name);
}
```

- Интерфейсы

```typescript
interface Person {
    name: string;
    age: number;
}

```

- Перечисления (enums)

```typescript
enum Color {Red, Green, Blue}
let c: Color = Color.Green;
```

- TypeScript расширяет возможности классов, добавляя модификаторы доступа (`public`, `private`, `protected`) и строгую типизацию свойств:

```typescript
class Animal {
    private name: string;
    
    constructor(name: string) {
        this.name = name;
    }
}

```

### Как преобразуются абстракции TypeScript в JavaScript

Перечисление:

```typescript
enum Direction {
    Up,
    Down,
    Left,
    Right
}
```

JavaScript:

```javascript
const Direction = {
  Up: 0,
  Down: 1,
  Left: 2,
  Right: 3
};
```

#### Интерфейсы

Интерфейсы в TypeScript служат исключительно для структурной типизации и проверки соответствия формы данных на этапе компиляции. При трансляции TypeScript в JavaScript интерфейсы полностью удаляются и не включаются в результат — в чистом JS они отсутствуют, так как JavaScript не поддерживает типы и интерфейсы на уровне выполнения. Это не нарушает целостность кода, потому что проверка типов является задачей компилятора, а в рантайме объекты остаются обычными JavaScript-объектами с нужными полями и методами.​

Когда интерфейсы используются в параметрах функций для полиморфизма, типы просто служат контрактом: компилятор проверяет, что объекты, передаваемые в функцию, имеют необходимую структуру. Например:

```typescript
interface Shape {
    area(): number;
}

function printArea(shape: Shape) {
    console.log(shape.area());
}
```

На этапе компиляции TypeScript гарантирует, что любой объект, передаваемый в printArea, реализует метод area. В скомпилированном JavaScript эта проверка отсутствует, функция просто вызывает shape.area() у переданного объекта. Полиморфизм достигается за счёт договорённости о структуре объектов, а не наличия интерфейсов в рантайме. Так TypeScript обеспечивает безопасность типов во время разработки без накладных расходов при выполнении

JavaScript:

```javascript
function printArea(shape) {
    console.log(shape.area());
}
```

## Использование кода в HTML

#### Простейший пример **встроенного кода**:
```html
<!DOCTYPE HTML>
<html>
<body>
    <p>Перед скриптом...</p>
    <script>
        alert('Привет, мир!');
    </script>
    <p>...После скрипта.</p>
</body>
</html>
```

~У тега script есть атрибут "language", который по умолчанию равен "JavaScript"

Тег `<script>` содержит JavaScript-код, который автоматически выполнится, когда браузер его обработает.

#### Пример внешнего кода (в отдельном файле .js):

Ссылка на локальный файл:

```html
<script src="/path/to/script.js"></script>
```

Ссылка на URL:

```html
<script src="https://...lodash.js"></script>
```

Если атрибут src установлен, то тело тега будет игнорироваться.
