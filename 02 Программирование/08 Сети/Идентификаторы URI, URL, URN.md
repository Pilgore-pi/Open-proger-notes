
Идентификаторы URI, URL и URN связаны между собой и служат для идентификации ресурсов в интернете, но у каждого есть свои особенности.

URI (Uniform Resource Identifier) — это общий унифицированный идентификатор ресурса, представляющий собой строку символов, которая однозначно определяет ресурс. URI может быть как адресом ресурса, так и его именем. 

URL (Uniform Resource Locator) — подтип URI, который не только идентифицирует ресурс, но и указывает способ доступа к нему и его местоположение. URL обычно содержит протокол (например, http, https, ftp), доменное имя и путь к ресурсу. Пример URL: https://www.example.com/index.html.

URN (Uniform Resource Name) — тоже подтип URI, который предназначен для уникального именования ресурса, но не даёт информации о местоположении или способе доступа к нему. URN независим от сетевого адреса и служит для постоянной идентификации ресурса, например, urn:isbn:978-3-16-148410-0 — уникальное имя книги по ISBN.

Таким образом, URI — это общее понятие, в рамках которого URL указывает на место ресурса, а URN — даёт ему уникальное имя, независимо от местоположения. Помимо этих, существуют и другие схемы URI для различных целей, но они реже встречаются.

Подробно:
- URI объединяет URL и URN.
- URL указывает, где ресурс находится и как получить к нему доступ.
- URN именует ресурс без указания его местоположения.

Это разделение важно для разных сценариев работы с ресурсами в интернете, например, для поиска по уникальному имени или прямого доступа по адресу.[2][3][4][5][6][7][8]
