* `"точный запрос"` — результаты должны содержать то что указано в кавычках
* `-слово` — результат не будет содержать указанные слова
* `запрос 1 OR запрос 2` или `|` — выдача результатов нескольких запросов
* `()` — оператор группировки
* `слово * слово` — любое слово вместо звездочки
* `слов*****` — поиск слова, начинающегося на "`слов`"
* `~слово` — поиск синонимов "слова" ==не работает==
* `2000..2002` — временной диапазон

>Регистр важен

>Пробел внутри оператора можно заменить на "`%20`" для корректного восприятия поисковиком

OR (именно `OR` или `|`, не `or` и не `или`) плохо работает, если результатов много. Сначала выдаются результаты последнего запроса в выражении, потом другие. Регистр влияет на результат

### Ключевые слова:

>Пробел после двоеточия недопустим

- `site:URL` — фильтрация результатов по сайту (несколько сайтов низя)
- `related:URL` — поиск страницы, связанные с указанным сайтом
- `link:URL` — поиск сайтов, в которых есть указанная ссылка
- `cache:` — ?
- `info:URL` — в результатах будет отображаться информация о сайте
- `filetype:type` — поиск файла указанного типа
- `date:(< | > | <= | >=)ГГГГ-ММ-ДД` — поиск по указанной дате (до, после)
- `date:ГГГГ*` — частично указанная дата
- `intext:слово или фраза` — указанная фраза должна содержаться в теле html-страницы
- `allintext` — явно можно использовать фразу с пробелами (аналогично intext)
- `intitle:слово или фраза` — указанная фраза должна содержаться в заголовке
- `allintitle:фраза` — аналогично intitle, можно использовать пробелы (хотя и в intitle можно `¯\_(ツ)_/¯`)
- `inurl:слово` — указанное слово должно содержаться в URL результата
- `location:"kyiv"` — поиск по местоположению. Примеры:
  - (Привести примеры москвы, Нью-йорка и т.д.)
- `lang:(en|ru|de|fr)`
	- en — английский
	- de — немецкий
	- fr — французский
- `imagesize:500x400` — поиск изображений заданного размера. В результатах будут отображаться не все изображения, соответствующие ограничению.

Дефисы в дате необязательны, но каким-то фигом результаты отличаются (лучше с дефисами). Оператор `date` не гарантирует, что будут выданы результаты указанной даты.

Можно использовать минус для поиска по сайту. Например `intext:ozon.ru -site:ozon.ru` — выполняет поиск сайтов, упоминающих озон, кроме самого озона.
