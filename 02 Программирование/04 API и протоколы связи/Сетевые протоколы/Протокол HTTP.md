> **HTTP (HyperText Transfer Protocol)** — текстовый протокол, основанный на транспортном протоколе TCP

HTTP поддерживает несколько методов запросов, каждый из которых имеет свое предназначение:

- **GET**: Запрашивает данные с сервера. Идемпотентный метод, повторные запросы не изменяют состояние сервера.
- **POST**: Отправляет данные на сервер для создания или обновления ресурса. Не является идемпотентным.
- **PUT**: Обновляет существующий ресурс на сервере. Является идемпотентным.
- **DELETE**: Удаляет ресурс с сервера. Также идемпотентный метод.
- **HEAD**: Запрашивает только заголовки ответа без тела.
- **OPTIONS**: Запрашивает поддерживаемые методы для ресурса

#### Структура HTTP-запроса и ответа

##### Запрос

HTTP-запрос состоит из следующих частей:

1. **Стартовая строка**: Содержит метод запроса, URL и версию протокола.

```http
GET /index.html HTTP/1.1
```

1. **Заголовки**: Дополнительная информация о запросе (например, тип контента).
2. **Тело**: Данные, отправляемые на сервер (в основном для методов POST и PUT).

Пример HTTP-запроса:

```http
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

#### Ответ

HTTP-ответ также имеет свою структуру:

1. **Стартовая строка**: Содержит версию протокола, статус-код и статусное сообщение.

```http
HTTP/1.1 200 OK
```

1. **Заголовки**: Дополнительная информация о ответе (например, тип контента).
2. **Тело**: Данные, возвращаемые сервером.

Пример HTTP-ответа:

```http
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>
    <body>
        <h1>Привет, мир!</h1>
    </body>
</html>
```

С переходом к HTTP/2 произошли значительные изменения в архитектуре протокола. В отличие от текстового формата HTTP/1.1, который использует простые строки для передачи данных, HTTP/2 использует бинарный формат, что позволяет улучшить производительность за счет мультиплексирования запросов и сжатия заголовков. Это означает, что клиент может отправлять несколько запросов одновременно по одному TCP-соединению без необходимости ожидания ответа на каждый из них
