Существует специальный текстовый файл, который используется системой Git, называемый `.gitignore`

Данный файл представляет собой простой текстовый файл, в котором могут быть перечислены пути к каталогам, файлам, шаблоны файлов и расширений, которые нужно игнорировать при выполнении действий в Git

Пример конфигурации `.gitignore`:

```gitignore
*.mp4
02 Программирование/01 .NET/C# 7.pdf
02 Программирование/01 .NET/Конспекты в тетради/
```

Здесь добавлены в список игнорирования все `.mp4` файлы, 1 файл формата `.pdf` и каталог `Конспекты в тетради`

> Нужно обратить внимание на то, что каталоги разделяются прямым слешем (`/`), а не обратным (`\`), и если шаблон задается для каталога, то в конце обязательно нужно ставить слеш `/`, чтобы обозначить каталог, так как без слеша в конце git может воспринимать шаблон как шаблон файла, у которого нет расширения 

> Все пути к файлам и каталогам должны указываться относительно файла `.gitignore`

> В файле `.gitignore` пути с пробелами **не нужно заключать в кавычки или экранировать специальными символами** - Git воспринимает пробел как часть имени файла или папки

#### Важные моменты:

- **Не используйте кавычки** вокруг путей с пробелами - это приведет к тому, что Git не распознает правило правильно[1](https://ru.stackoverflow.com/questions/432432/%D0%9D%D0%B5-%D0%B8%D0%B3%D0%BD%D0%BE%D1%80%D0%B8%D1%80%D1%83%D1%8E%D1%82%D1%81%D1%8F-%D1%84%D0%B0%D0%B9%D0%BB%D1%8B-%D0%B2-gitignore).
    
- Если в имени файла или папки есть специальные символы (например, `*`, `[`, `]`), их можно экранировать обратной косой чертой `\`, но для пробелов это не требуется[4](https://www.atlassian.com/ru/git/tutorials/saving-changes/gitignore).
    
- Если возникают сложности с пробелами при использовании команд Git (например, `git rm` с путями, содержащими пробелы), в командной строке путь нужно брать в кавычки или экранировать пробелы, но в `.gitignore` - нет[1](https://ru.stackoverflow.com/questions/432432/%D0%9D%D0%B5-%D0%B8%D0%B3%D0%BD%D0%BE%D1%80%D0%B8%D1%80%D1%83%D1%8E%D1%82%D1%81%D1%8F-%D1%84%D0%B0%D0%B9%D0%BB%D1%8B-%D0%B2-gitignore).

Чтобы просмотреть список игнорируемых файлов, нужно выполнить команду:

```git
git ls-files --others -i --exclude-standard
```

Эта команда выведет список всех игнорируемых файлов и папок в текущем каталоге и подкаталогах

Синтаксис `.gitignore`:

- `#` — однострочный комментарий. Чтобы экранировать символ `#`, нужно поставить `\` перед ним
- `<пустая-строка>` — игнорируется
- 

- A leading "`**`" followed by a slash means match in all directories. For example, "`**/foo`" matches file or directory "`foo`" anywhere, the same as pattern "`foo`". "`**/foo/bar`" matches file or directory "`bar`" anywhere that is directly under directory "`foo`".
    
- A trailing "`/**`" matches everything inside. For example, "`abc/**`" matches all files inside directory "`abc`", relative to the location of the `.gitignore` file, with infinite depth.
    
- A slash followed by two consecutive asterisks then a slash matches zero or more directories. For example, "`a/**/b`" matches "`a/b`", "`a/x/b`", "`a/x/y/b`" and so on.
    
- Other consecutive asterisks are considered regular asterisks and will match according to the previous rules.

## [Исключение локальных файлов без создания файла _.gitignore_](https://docs.github.com/ru/get-started/git-basics/ignoring-files#excluding-local-files-without-creating-a-gitignore-file)

Если вы не хотите создать файл для совместного использования файла `.gitignore` с другими пользователями, можно создать правила, которые не будут попадать в коммит в репозитории. Этот метод можно использовать для локально создаваемых файлов, которые не должны создавать другие пользователи, например, для файлов, создаваемых редактором.

Используйте любимый текстовый редактор, чтобы открыть файл, вызываемый `.git/info/exclude` в корневом каталоге репозитория Git. Любое добавляемое здесь правило не будет синхронизироваться, и будут игнорироваться только файлы для локального репозитория.

1. Откройте GIT Bash.
2. Перейдите к расположению репозитория Git.
3. Откройте файл `.git/info/exclude`с помощью текстового редактора.

> Пути в файле `exclude` прописываются относительно корня репозитория, так же, как и в `.gitignore`
