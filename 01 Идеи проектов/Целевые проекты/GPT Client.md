
Создать бота-оболочку, который использует GPT через API. Бот должен быть на сайте или в Android приложении (и decktop)

* **API ключ** должен предоставить сам пользователь. Это позволит не зависеть от способов оплаты и предоставления доступа из приложения. Доступ предоставляют сторонние сервисы, а пользователь лишь предоставляет ключ.
* **Контекстные подсказки**. Добавить возможность делать шаблоны промптов при начале диалога.
* **Параметры**: Температура, РазмерКонтекста, ИспользоватьКонтекстДиалога, Озвучивание ответа
* **Аудиоуправление**. Нажал кнопку — говоришь. Речь синтезируется и бот воспринимает. Ответ бота также озвучивается. Типа Джарвиса.
* **Информация по аккаунту**. Предоставлять информацию об акке, которую можно получить из АПИ ключа
* **Распознавание текста** на изображении или документе с помощью API (`#<Команда>#` или `[<Команда>]`)
* **Запланированные запросы**. Сделать возможность добавления запросов, которые будут выполнятся автоматически в определенное время (структура: Название, промпт, ответ). Если прога не работает в указанное время, то запрос должен выполниться как только появится такая возможность. ЧТобы не накапливалось много запросов, нужно ограничить их количество и не позволять накопливать одни и те же запросы.

Задать начальный промпт, сказать боту, что у него будет такое-то имя. То есть первое сообщение будет системным.

Создать роли, например, гопота, слуга, раб, раздраженный... Роли зависят от начальных скрытых промптов

Добавить предупреждения, если кончаются токены, вопросы и т. д., если будет ограниченный тариф

>Озвучка: можно использовать нейросеть SteosVoice для озвучки любого текста ИЛИ синтезатор речи

## Команды

Это специализированные ключевые слова, которые выполняют специальные скрипты:

* Перевод по API
* Конвертация данных по API (Convertio)
* OCR (распознавание текста) (Convertio...)
* Инфа о погоде
* Выполнение действий системного администрирования (выключить комп через полчаса, перезагрузить комп, заблокировать систему (окно входа в систему)...)
* Можно создавать свои команды

>Команды могут быть написаны не полностью, если пользователь ввел достаточно символов для однозначной идентификации команды (`#Пого#`, `#Погода#`).

>Разделители между словами команды могут быть разными, но одинаковыми: (`#Погода в Азове`). Все возможные разделители: ` `, `_`, `    `, `-`, `+`, `—`, `–`, `=`, `&`, `*`, `/`, `\`, `|`, `.`

При вводе спецсимвола `#` делать автозамену на `##`. Учесть, что нужно делать автозамену, если перед спецсимволом нет такого же спецсимвола (по аналогии со скобками в Obsidian).
