ОПИСАТЬ, КАКИЕ БРАУЗЕРЫ ОТ КАКИХ ДВИЖКОВ НАСЛЕДУЮТСЯ (ДЕРЕВО)

> **Веб-браузеры** — это специализированное программное обеспечение, предназначенное для просмотра веб-сайтов и работы с интернет-ресурсами, разработанное в 1989 году Тимом Бернерсом-Ли.

Первый браузер назывался **WorldWideWeb** и был создан как часть системы для обмена информацией в CERN

Браузер является:

* Клиентским приложением
* Кроссплатформенным (большинство современных браузеров)
* Многопроцессным
* Интерпретирующим (для веб-кода)
* Многофункциональным:
    * Средством просмотра веб-страниц
    * HTTP-клиентом
    * Интерпретатором кода (HTML, CSS, JavaScript)
    * Менеджером кэша и куки
    * Средством для работы с медиа
* Приложением общего назначения для работы в интернете

## Основные компоненты браузера

### Пользовательский интерфейс (UI)

* Адресная строка (omnibox)
* Кнопки навигации (назад, вперед, обновить, стоп)
* Закладки и история
* Меню и панели инструментов
* Вкладки и окна

### Движок рендеринга (Rendering Engine)

Отвечает за отображение веб-страниц:
* **Webkit** — используется в Safari, Chrome (ранее)
* **Blink** — используется в Chrome, Edge, Opera
* **Gecko** — используется в Firefox
* **Trident** — использовался в Internet Explorer (устарел)

### JavaScript движок (JS Engine)

Интерпретирует и выполняет JavaScript код:
* **V8** — Chrome, Edge, Node.js
* **SpiderMonkey** — Firefox
* **JavaScriptCore** — Safari
* **Chakra** — Internet Explorer (устарел)

### Сетевой модуль (Network Module)

* Управление HTTP/HTTPS запросами
* Кэширование данных
* Управление соединениями
* Обработка протоколов

### Хранилище данных (Storage)

* Локальное хранилище (LocalStorage)
* Сессионное хранилище (SessionStorage)
* Cookies
* IndexedDB
* WebSQL (устарел)

## Как работает браузер

### Этап 1: Ввод URL и разрешение имени хоста

```
1. Пользователь вводит URL в адресную строку
2. Браузер проверяет кэш DNS
3. Если кэша нет, отправляется запрос к DNS серверу
4. DNS возвращает IP-адрес сервера
```

### Этап 2: Установление соединения

```
1. Браузер устанавливает TCP соединение (three-way handshake)
2. Если используется HTTPS, выполняется TLS handshake
3. Соединение готово к передаче данных
```

### Этап 3: Отправка HTTP запроса

```
1. Браузер формирует HTTP запрос (GET, POST и т.д.)
2. Добавляет заголовки (User-Agent, Accept, Cookies и т.д.)
3. Отправляет запрос на сервер
```

### Этап 4: Получение ответа

```
1. Сервер обрабатывает запрос
2. Возвращает HTML документ с кодом состояния (200, 404, 500 и т.д.)
3. Браузер начинает получать данные
```

### Этап 5: Парсинг HTML (Parsing)

```
1. Браузер построчно читает HTML документ
2. Создает DOM дерево (Document Object Model)
3. При встрече с <link> для CSS или <script> для JS:
   - Загружает эти ресурсы
   - Может блокировать парсинг (для скриптов)
```

### Этап 6: Загрузка ресурсов

```
1. Браузер обнаруживает ссылки на внешние ресурсы:
   - Изображения (<img>)
   - Стили (<link>)
   - Скрипты (<script>)
   - Шрифты (@font-face)
   - Видео и аудио
2. Загружает их параллельно (обычно 6-8 соединений на домен)
```

### Этап 7: Построение CSSOM

```
1. Браузер парсит CSS файлы
2. Создает CSSOM дерево (CSS Object Model)
3. Объединяет DOM и CSSOM в Render Tree
```

### Этап 8: Компоновка (Layout)

```
1. Браузер вычисляет позицию и размер каждого элемента
2. Учитывает viewport, медиа-запросы, flexbox, grid
3. Создает макет страницы
```

### Этап 9: Рисование (Painting)

```
1. Браузер преобразует Render Tree в пиксели
2. Рисует текст, изображения, границы, фоны
3. Создает слои (layers) для оптимизации
```

### Этап 10: Композиция (Compositing)

```
1. Браузер объединяет слои в правильном порядке
2. Применяет эффекты (тени, трансформации, прозрачность)
3. Отправляет финальное изображение на GPU
4. Выводит на экран
```

### Этап 11: Выполнение JavaScript

```
1. Браузер выполняет JavaScript код
2. Может модифицировать DOM и CSSOM
3. Вызывает события (load, DOMContentLoaded и т.д.)
4. При изменении DOM/CSS запускается переотрисовка
```

## Безопасность в браузерах

* **Same-Origin Policy** — ограничение доступа между разными источниками
* **Content Security Policy (CSP)** — защита от XSS атак
* **HTTPS/TLS** — шифрование передачи данных
* **Sandbox** — изоляция вкладок и фреймов
* **Управление permissions** — доступ к камере, микрофону, геолокации
