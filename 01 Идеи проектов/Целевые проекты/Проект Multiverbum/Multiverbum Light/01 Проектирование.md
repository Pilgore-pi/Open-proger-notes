
#### Суть

Создание простеньких тестов на запоминание и для практики всевозможных тем. Функционал универсален, поэтому можно будет использовать его для изучения любых областей знаний

#### Объекты БД

Тест — JSON файл (исходные readonly данные):

```json
{
    "Title" : "test title",
    "Description" : "desc", /*can be null*/
    "Attempts" : 0,
    "LastAttemptScores" : 0, /* N/M */
    "AverageScores" : 0, /* N/M */
    "Topic" : "English/Anime/Attack on Titan", /*topic & subtopics*/
    "LastSectionIndex" : -1,
    "AllowPageNavigationBackwards" : false,
    "AllowPageNavigationForwards" : true,
    /*Auto creation date; nothing in JSON*/
    
    "Sections" : [{
        "Prompt" : "text",
        "AnswerOptions" : [ "option1", "option2" ], /*can be null*/
        "CorrectAnswers" : [ "ans1", "ans2", "ans3" ], /*can be null*/
        "SemiCorrectAnswers": null,
        "Priority" : 100 /*percents; by default*/
    }]
}
```

> СТОП, нахуя мне JSON, если создание будет через UI? В JSON формате можно обмениваться HTTP запросами между сервером и клиентом

Данные загружаются в БД и там уже с ними можно делать что угодно

### Функционал

- Использование, редактирование, удаление и создание тестов
- Сбор и отображение статистики по прохождению тестов
- Возможность прерваться, а потом вернуться к незавершенной сессии
- Отображать вопросы по приоритету (который рассчитывается в зависимости от частоты правильных ответов)
- Поддержка 2 языков: английский и русский (использовать словари ресурсов или как их там)
- Пасхалка: 3 язык — при нажатии на кнопку смены языка 10 раз, будет выбран русский язык в абсурдном и мемном стиле (чтобы не испортить пасхалку, сделать кнопку недоступной на пару секунд после нахождения пасхалки). При нахождении пасхалки издать смешной звук (ГОДЗИЛЛА) и сменить тему

### GUI

- ***Start page***: тесты, сгруппированные по темам и подтемам (создать ЭУ "иерархический список")
    - Test element: основная инфа + кнопки управления (удалить, редактировать, копировать, начать, \[продолжить\])
    - Кнопка создания теста
- ***Editing page*** (or creation): иерархическое представление данных с раскрываемыми группами
    - Название, Описание, ...
    - Задания
        - Описание (обяз.)
        - Приоритет (ползунок) (по умол. 100%)
        - Варианты ответов (необ.)
            - Текст (обяз.)
        - Правильные ответы (обяз.)
            - Текст (обяз.)
        - Частично правильные ответы (необ.)
            - Текст (обяз.)
        - Кнопка **"удалить"**
        - Кнопка **"дублировать"**
    - Кнопка **"Домой"** с предупреждением при нажатии ("Вы не завершили тест, вы уверены, что хотите прервать его? Изменения будут сохранены")
    - Кнопка
- ***Using page***: прохождение теста
    - Кнопка **"Домой"** с предупреждением при нажатии ("Вы не завершили тест, вы уверены, что хотите прервать его? Изменения будут сохранены")
    - \[Кнопка **"Завершить"**\] на последнем вопросе => переход на страницу **Results page**
    - Кнопка **"Показать правильный ответ"** автоматически не засчитывает ответ на вопрос. Подумать над этим
- ***Results page***: Показаны все ответы пользователя и правильные ответы в таблице
- ***Statistics page***: Статистика, показанная в таблице
- ***Warning window***: Окно с предупреждением, отображаемое при попытке перехода на главную страицу во время прохождения теста

Глобальные кнопки (которые есть на каждой странице):

- "Домой" — возвращает на главную страницу (предупреждение)
- "Статистика" — открывает страницу со статистикой поверх текущей страницы
- "Назад" — ссылка на предыдущую страницу (не всегда активна)
- "Вперед" — переход на следующую страницу (не всегда активна)
