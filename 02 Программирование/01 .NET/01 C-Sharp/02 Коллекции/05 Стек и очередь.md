
## Очередь

> Класс `Queue<T>` представляет собой структуру данных “очередь”, работающую по принципу FIFO (First In First Out). Для очереди, как структуры данных, разрешены операции добавления в конец и удаления из начала.

При создании объекта Queue можно указать начальную емкость или массив, как для класса `List<T>`.
Свойство `Count` отображает количество элементов

| Метод                      | Описание                                           |
| :------------------------- | :------------------------------------------------- |
| `Enqueue(T item)`          | Добавление элемента                                |
| `Dequeue()`                | Удаляет элемент и возвращает его                   |
| `TryDequeue(out T result)` | Попытка удалить элемент                            |
| `Peek()`                   | Возвращает первый элемент                          |
| `TryPeek(out T result)`    | Попытка вернуть первый элемент                     |
| `Contains(T item)`         | Проверяет, содержится ли данный объект в коллекции |
| `Clear()`                  | Очищает коллекцию                                  |

## Стек

Данный класс реализует структуру данных «стек», работающую по принципу LIFO (Last In First Out). Для стека, как структуры данных, определены операции добавления в конец, удаление из конца, просмотр последнего элемента.
Количество элементов содержит свойство Count.

| Метод        | Описание                                           |
| :----------- | :------------------------------------------------- |
| `Push()`     | Добавляет элемент в начало                         |
| `Pop()`      | Удаляет последний элемент                          |
| `TryPop()`   | Попытка удалить последний элемент элемент          |
| `Peek()`     | Возвращает последний элемент                       |
| `TryPeek()`  | Попытка вернуть последний элемент                  |
| `Contains()` | Проверяет, содержится ли данный объект в коллекции |
| `Clear()`    | Очищает коллекцию                                  |
