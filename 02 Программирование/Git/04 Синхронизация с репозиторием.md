
Для синхронизации локального репозитория с удаленным репозиторием используются, в основном, эти команды:

| Команда Git                                                                  | Описание                                                                                                                                                                                  |
| ---------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`git clone <repository>`](https://git-scm.com/docs/git-clone)               | выполняет копирование всего репозитория в текущий каталог (каталог, в который был осуществлен переход командой `cd` (Change Directory))                                                   |
| [`git pull <repository>`](https://git-scm.com/docs/git-pull)                 | выполняет загрузку и автоматическое слияние изменений из удаленного репозитория                                                                                                           |
| [`git push <repository> [<branch-name>]`](https://git-scm.com/docs/git-push) | выполняет отправку изменений в удаленный репозиторий                                                                                                                                      |
| `git remote [-v]`                                                            | отображает список всех настроенных удаленных репозиториев. Флаг `-v` говорит о том, что нужно выводить адреса для чтения и записи, привязанные к репозиториям                             |
| `git remote add <shortname> <URL>`                                           | Добавляет ссылку на удаленный репозиторий в текущий локальный репозиторий. При этом нельзя добавлять уже добавленные ссылки                                                               |
| `git remote show`                                                            | отображает основную информацию об удаленном репозитории: URL репы, локальные и общие ветки и др. Если репозиториев несколько, то будет показана информация по всем удаленным репозиториям |
| `git remote rename <old-name> <new-name>`                                    | выполняет переименование удаленного репозитория                                                                                                                                           |
| `git remote remove <repository>`                                             | удаление удаленного репозитория                                                                                                                                                           |
| `git fetch <repository>`                                                     | получение изменений из репозитория                                                                                                                                                        |

> Для выполнения некоторых операций требуются специальные права, например, чтобы выполнить `push`, нужно иметь права на запись
