> **HTML (Hyper Text Markup Language)** — это [[01 Парадигмы языков программирования#Декларативное программирование|декларативный]] язык разметки гипертекста, предназначенный для простой организации элементов графического пользовательского интерфейса (GUI) в контексте браузера

> **Браузер** — это среда (движок), которая анализирует HTML код и отображает элементы интерфейса согласно HTML документу

- Язык **[[02 Введение в CSS|CSS]]** позволяет очень гибко настроить GUI, беря за основу компоновку элементов, определенную через HTML.

- Языки *front-end* **[[01 Введение в JS|JavaScript]]** и **TypeScript** используются, в основном, для обработки логики взаимодействия пользователя с графическими элементами. Также может использоваться для реализации полноценной логики веб-приложения, основанного на HTML, CSS, JS/TS

- Для написания логики сложных веб-приложений и простых сайтов используются *back-end* языки программирования (Python, C#, Go, Ruby, PHP...)

HTML код основан на стандарте XML, поэтому в HTML используются **теги**. У тегов могут быть атрибуты (свойства) и подчиненные теги.

```html
<!-- Объявление HTML документа с русским языком -->
<html lang="ru"></html>

<!-- Полностью аналогично: -->

<html>
    <lang>ru</lang>
</html>
```

> HTML не чувствителен к регистру текста, но согласно общепринятым стандартам используется, преимущественно, нижний регистр

Теги бывают 2 видов:

1. **Парные** `<tag></tag>` или сокращенная версия: `<tag/>`
2. **Одинарные** `<br>`

- Теги всегда должны заключаться в угловые скобки и записываться латинскими символами

Документы HTML могут иметь следующие расширения файлов:

- `.html` — классический вариант
- `.htm` — аналогичный вариант, используемый в старых ОС, в которых было ограничение на длину расширения файла
- `.phtml` — файл с PHP кодом; расширение явно обозначает, что PHP код может содержать HTML вставки
- `.php` — файл с кодом PHP; PHP может содержать HTML код

##### Парсинг HTML браузером

По мере развития стандарта HTML создавались новые теги, но несмотря на это старые теги, как и новые доступны в любой версии HTML документа.
Это сделано для поддержки обратной совместимости.

В *HTML 5* были добавлены одни теги и удалены другие, но их все равно можно использовать, так как анализатор текста HTML в браузере поддерживает все версии

> Множество подряд идущих пробельных символов (пробелы, переводы строк) отображаются на странице как один пробел.

> Если тег не определяется парсером, то он игнорируется

### Основные теги

| Тег                           | Описание                                                                                                                                                            |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<!DOCTYPE html>`             | определение типа документа и версии HTML                                                                                                                            |
| `<html/>`                     | корневой элемент HTML документа. Все теги должны находиться внутри этого тега                                                                                       |
| `<!-- -->                   ` | многострочный комментарий (не рекомендуется использовать) ПАЧЕМУ                                                                                                    |
| `<head/>`                     | область метаданных, предназначенная для программной обработки, которая определяет используемые стили, заголовки, кодировку и т. д.                                  |
| `<meta/>`                     | представляет метаданные, которые не могут быть описаны другими тегами (`<base>`, `<link>`, `<script>`, `<style>`, `<title>`)                                        |
| `<body/>`                     | обозначает тело HTML документа                                                                                                                                      |
| `<main/>`                     | главная часть тела (`body`) документа; данная секция должна содержать основную функциональность                                                                     |
| `<title/>`                    | определяет заголовок страницы, отображаемый в заголовке окна браузера                                                                                               |
| `<style/>`                    | определяет визуальное оформление описанное на CSS                                                                                                                   |
| `<link/>`                     | определяет внешнюю зависимость (ссылку)                                                                                                                             |
| `<header/>`                   |                                                                                                                                                                     |
| `<footer/>`                   | нижний колонтитул документа или секции                                                                                                                              |
|                               | **Теги** **компоновки**                                                                                                                                             |
| `<div/>`                      | *блочный* контейнер для группировки элементов и применения стилей; [block-level content](https://developer.mozilla.org/en-US/docs/Glossary/Block-level_content)     |
| `<span/>`                     | *строковый* контейнер для группировки элементов и применения стилей; [inline-level content](https://developer.mozilla.org/en-US/docs/Glossary/Inline-level_content) |
| `<section/>`                  | автономный раздел документа, который может содержать заголовки и другие элементы; `section` всегда должен иметь заголовок                                           |
| `<base/>`                     | содержит корневой адрес URL, используемый как якорь для всех относительных URL в документе; HTML документ может содержать только один тег `<base/>`                 |

| Тег           | Описание                                                                                                                                                                                                                   |

## Спецсимволы

Формат спецсимволов:
`&name` или `&#NNN`, где `NNN` — код символа Unicode (десятеричный), `name` — зарезервированное слово из справочника

Методы верстки:

* Табличный — каркас сайта конструируется с помощью HTML таблиц. Это единственный тип верстки, который вообще не требует CSS. Остальные типы верстки требуют его.

Рекомендуемые типы изображений:

* bmp — никогда
* jpg — сжатие с потерями
* png — сжатие без потерь
* gif — устаревший; сжатие без потерь, но 256 цветов; анимация
* svg — текстовый файл, для отрисовки изображений, распространенный web-формат.
