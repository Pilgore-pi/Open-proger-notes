## Массивы

```js
// Массивы
let arr = new Array();
let three = new Array(1, 2, 2);
let four = new Array(4); // 4 пустых элемента
let arr1 = [];
let arr2 = [1, 2, 3];
arr2[0] = arr2[2];

// добавление
arr2[3] = 4;
```

Если инициализировать элемент по индексу, который больше длины массива, то массив станет длиной равной индексу, а все промежуточные элементы будут undefined.

| Свойство / Метод | Описание |
|----------------- |- 
| `.length` | длина массива
| `at()` | элемент по индексу. Позволяет получать элементы считая с конца. Для этого используются отрицательные `индексы
| `sort()` | сортировка
| `reverse()` |
| `pop()` | удаляет последний элемент
| `shift()` | удаляет первый элемент
| `unshift()` | добавляет один или несколько элементов в начало
| `splice()` | добавляет или удаляет элементы
| `concat()` | соединение с другим массивом
| `indexOf()` | индекс первого вхождения указанного элемента
| `lastIndexOf()` | индекс последнего вхождения
| `forEach()` | вызывает функцию для каждого элемента
| `every()` | true, если каждый элемент удовлетворяет условию
| `some()` | true, если хотя бы 1 элемент удовлетворяет условию
| `map()` | аналог LINQ Select()
| `toString()` | строковое представление

Интерпретатор JS оптимизирован под использование упорядоченных однотипных объектов в массиве, поэтому стоит избегать использования разных типов в массиве

> Длину массива можно менять вручную. Если укоротить заполненный массив, то будут утеряны данные. Таким образом самый простой способ очистить массив: `arr.length = 0`

## Многомерные массивы

```js
let matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
];

alert(matrix[1][1]); // 5, центральный элемент
```

При суммировании списка с числом список преобразуется в строку

```js
alert("" + 1);    // "1"
alert("1" + 1);   // "11"
alert("1,2" + 1); // "1,21"
```

Массивы бессмысленно сравнивать через `==`, так как они имеют разные ссылки

```js
alert([] == []);   // false
alert([0] == [0]); // false
```
