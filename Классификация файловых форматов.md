## Текст

- **Обычные техстовые форматы**
    - `.txt` — неформатированный текст с заданной кодировкой
    - `.rtf` (Rich Text Format) — расширенный текстовый формат от Microsoft. Поддерживает форматирование текста, таблицы и изображения, используя специальные управляющие коды. Совместим с множеством текстовых редакторов (например, Microsoft Word, LibreOffice)
    - `.md` — текст, форматируемый с помощью синтаксиса Markdown
    - `.prn` — формат предназначенный для передачи данных на печать. Обычно представляет собой текстовый файл с управляющими символами для принтеров
- **Форматы файлов с кодом**
    - `.c` — C
    - `.cpp` `.cxx` `.h` — C++
    - `.cs` — C#
    - `.java` — Java
    - `.py` `.pyw` — Python
    - `.js` `.jsx` — JavaScript
    - `.ts` — TypeScript
    - `.rs` — Rust
    - `.dart` — Dart
    - `.kt` — Kotlin
    - `.scala` — Scala
    - `.bsl` — 1С:Предприятие
    - `.ps1` — PowerShell
    - `.bat` — Batch script
    - `.sql` — SQL
- **Текстовые документы**
    - `.pdf` (Portable Document Format) — универсальный формат электронных документов, разработанный компанией Adobe в 1993 году. Он предназначен для сохранения внешнего вида документа, включая текст, шрифты, графику и другие элементы, независимо от операционной системы или программного обеспечения. Документ сохраняет свой оригинальный вид на любых устройствах. Возможно добавление паролей и ограничений на редактирование
    - `.odt` — открытый формат, представляющий ZIP-архив, содержащий XML-документ, который описывает текст, стили, изображения и другие элементы. Более ограниченные возможности форматирования
    - **Форматы MS Word**
        - `.doc` — документ Word 1997-2003 годов, двоичный формат
        - `.docx` — документ Word, ZIP-архив с XML-разметкой
        - `.docm` — документ Word с поддержкой макросов
        - `.dot` — шаблон документа 1997-2003 годов
        - `.dotx` — шаблон документа
        - `.dotm` — шаблон документа с поддержкой макросов
    - **Веб-страницы**
        - `.html` (Hyper Text Markup Language) — файл веб-страницы HTML
        - `.htm` (Hyper Text Markup) — этот формат возник из-за ограничений некоторых старых операционных систем, которые не поддерживали расширения файлов длиной более трех символов. В остальном формат идентичен `.html`
        - `.mhtml` — (Multipurpose Internet Mail Extensions (MIME) HTML) — архив веб-страниц с HTML кодом и со всеми связанными ресурсами (изображения, CSS, JS...). Позволяет пользоваться веб-страницами оффлайн
        - `.mht` — история аналогична `.htm`
- **Электронные книги**
    - `.fb2` (FictionBook) — разработан в России и представляет собой XML-документ, предназначенный для хранения художественной литературы
    - `.djvu` — формат, разработанный для хранения отсканированных документов и изображений с высоким качеством при низком размере файла
    - `.epub` (Electronic Publication) — открытый стандарт для электронных книг, который поддерживается большинством современных приложений. Формат основан на HTML и CSS
    - `.mobi` — формат, разработанный компанией Mobipocket. Он используется преимущественно на устройствах Amazon Kindle

## Массивы данных

- **Бинарные данные**
    - `.bin` — общий бинарный формат, обычно не имеет заголовка
    - `.dat` — файл с нерегламентированной структурой. Может хранить любые данные с любой структурой
    - `.iso` — файл образа CD/DVD диска, который содержит всю информацию диска
    - `.dll` (Dynamic Link Library) — хранит код и данные, которые могут быть использованы несколькими программами одновременно. Используется в Windows. Файлы `.dll` имеют формат Portable Executable (PE), аналогичный `.exe` файлам, и могут содержать секции кода, данных и ресурсов. Использование .dll файлов позволяет разработчикам разделять код на модули, что облегчает поддержку и развитие программ. Это также способствует экономии ресурсов, так как одна библиотека может использоваться несколькими приложениями одновременно
    - `.so` — Unix аналог DLL
- **Архивы**
    - `.zip` — поддерживает сжатие и простое объединение файлов. Алгоритм сжатия `Deflate`
    - `.rar` — предназначен для более эффективного сжатия по сравнению с `ZIP`. Используются различные алгоритмы сжатия (`LZ77`). Поддерживает создание самораспаковывающихся архивов. Позволяет восстанавливать поврежденные архивы
    - `.7z` — формат программы-архиватора 7-Zip. Используемые алгоритмы: `LZMA` и `LZMA2`. Поддержка шифрования `AES-256`
    - `.exe` — данный формат можно использовать для самораспаковывающихся архивов
    - `.msi` (Microsoft Installer) 
    - `.jar` — формат архива, используемый в Java для упаковки классов и ресурсов в одном файле. Основан на ZIP
    - `.tar` — аналог ZIP для UNIX
    - `.gzip` — формат исключительно для сжатия файлов (алг `Deflate`). Часто используется в веб-разработке для сжатия HTML, CSS и JavaScript-файлов перед их отправкой на сервер
    - `.sit` — формат архива, используемый в MacOS. Используются различные алгоритмы
- **Электронные таблицы (spreadsheets)**
    - `.csv` (Comma-Separated Values) — текстовый формат, предназначенный для представления табличных данных. Каждая строка в файле соответствует строке таблицы, а поля в строке разделяются запятыми или другими символами. Если поле содержит зарезервированные символы (например, запятую или новую строку), оно должно быть заключено в двойные кавычки. Если в поле есть двойные кавычки, они представляются двумя кавычками подряд
    - `.dif` (Data Interchange Format) — текстовый формат, созданный для обмена данными между программами. Обладает бОльшими возможностями по сравнению с CSV. В заголовке файла содержится инфа о структуре и параметры описания данных 
    - `.slk` (symbolic link) — формат от Microsoft, упрощенный аналог `.xls`. Каждая строка данных начинается с символа, указывающего тип данных (например, `C` для ячеек, `P` для форматов и т.д.). Строки в файле представляют собой записи таблицы, а данные разделяются специальными символами
    - **MS Excel**
        - `.xls` — формат Excel 1997-2003 годов
        - `.xlsx` — основной формат без поддержки макросов
        - `.xlsm` — книга Excel с поддержкой макросов
        - `.xlsb` — двоичная кника Excel
        - `.xla` — надстройка Excel 1997-2003 годов
        - `.xlam` — надстройка Excel
        - `.xlt` — шаблон Excel 1997-2003 годов
        - `.xltx` — шаблон Excel

## Исполняемые файлы

- `.exe` — наиболее распространенный формат исполняемых файлов в операционных системах *Windows*. Он содержит двоичный код, который может быть выполнен непосредственно операционной системой
- `.pe` (Portable Executable) — расширенный `.exe` формат
- `.msi` — формат пакета установки Windows Installer, который используется для установки, удаления и обслуживания программного обеспечения
- `.bat` — пакетный файл, содержащий последовательность команд, которые выполняются в командной строке (для Windows и DOS)
- `.com` — простой формат исполняемого файла, используемый в MS-DOS и Windows. Он не имеет заголовка и ограничен 64 КБ
- `.out` — один из первых форматов исполняемых файлов в UNIX-системах
- `.elf` (Executable and Linkable Format) — cтандартный формат исполняемых файлов для UNIX систем. Является более современным форматом, по сравнению с `.out`
- `.coff` (Common Object File Format) — aормат исполняемых файлов, который используется в некоторых UNIX-системах и поддерживает разделяемые библиотеки

## Изображения

        - `.tiff` (Tagged Image File Format) — поддерживает многослойные изображения, настраиваемое качество. Может иметь сжатие, а может и не иметь. Алгоритмы сжатия: `LZW`, `PackBits`

        - `.webp` — поддерживает сжатие с потерями и без потерь. Поддерживает *прозрачность*. На 30% *лучшее сжатие* по сравнению с PNG, JPEG. Поддерживает *анимацию* разработан Google в 2010 году

## Аудио

- Без сжатия
    - `.wav` (Waveform Audio File Format) — стандартный формат для хранения аудио на Windows-платформах, разработанный Microsoft и IBM.
    - `.aiff` (Audio Interchange File Format) — разработан компанией Apple и используется для хранения высококачественного аудио. Около 10МБ на 1 минуту аудио
- Сжатие без потерь (`lossless`)
    - `.flac` (Free Lossless Audio Codec) — *открытый* формат, который сохраняет оригинальное качество звука, но при этом уменьшает размер файла на 30-60%
    - `.ape` (Monkey's Audio) — формат менее распространен, чем `.flac`
- Сжатие с потерями (`lossy`)
    - `.mp3` (MPEG Audio Layer III) — *проприетарный* формат <a href="https://www.youtube.com/watch?v=z2EUT4gwkr4&t=0s">Побединский</a>
    - `.ogg` (Ogg Vorbis) — *открытый* аудио формат, аналог `.mp3`, но с более продвинутой технологией сжатия Vorbis. Формат предлагает лучшее качество по сравнению с MP3, но немного больший размер файла.
- **Специальные**
    - `.midi` (Musical Instrument Digital Interface) — не хранит аудиоданные, а представляет собой набор команд для музыкальных инструментов (например, синтезаторов)

## Видео

- `.mp4` (MPEG-4 Part 14) — один из самых популярных форматов для хранения видео и аудио данных. Он поддерживает множество кодеков и обеспечивает хорошее качество при относительно небольшом размере файла
- `.mov` — формат Apple с высоким качеством и большим весом. Часто применяется в видеоредакторах
- `.wmv` (windows media video) — формат, разработанный Microsoft для потокового вещания и хранения видео
- `.avi` (Audio Video Interleave) — формат, разработанный Microsoft в 1990-х годах. Устарел и может не поддерживать современные кодеки. Неэффективные алгоритмы сжатия
- `.mkv` (Matroska Video) — российский открытый формат
- `.flv` (Flash Video) — формат, разработанный Adobe для потоковой передачи видео через Flash Player. Хотя он был популярен для веб-видео, его использование снизилось после отказа от Flash
- `.webm` — открытый формат для от Google, разработанный в 2010 году, поддерживается в HTML5
- `.3gp` — Формат для мобильных устройств с низким битрейтом; часто использовался для записи видео на телефоны

# Кодеки

> **Кодек** — это программный или аппаратный инструмент, используемый для сжатия и распаковки аудио- и видеофайлов. Название "кодек" является сокращением от "coder/decoder", и он играет ключевую роль в обработке мультимедийных данных

Аудио и видео кодируется кодеком для экономии места. При воспроизведении кодек декодирует файл.

#### Аудиокодеки

**Сжатие без потерь (lossless)**

- FLAC (Free Lossless Audio Codec) — Популярный формат для хранения аудио без потерь
- ALAC (Apple Lossless Audio Codec) — Аналог FLAC, разработанный Apple

**Сжатие с потерями (lossy)**

- **MP3** — Один из самых распространенных аудиокодеков, используемый для сжатия музыкальных файлов
- **AAC** (Advanced Audio Coding) — Более современный аудиокодек, который обеспечивает лучшее качество звука при том же битрейте по сравнению с MP3
- **Ogg Vorbis**

##### Bluetooth кодеки

- **SBC** — стандартный кодек для Bluetooth-устройств, обеспечивает базовое качество звука
- **aptX** — Кодек с более высоким качеством звука по сравнению с SBC, используется в некоторых устройствах Android
- **LDAC** — Кодек от Sony для передачи высококачественного звука по Bluetooth

#### Видеокодеки

Lossless кодеки для видео используются редко из-за больших размеров файлов

**Lossy кодеки:**

- **H.264** — Один из самых популярных видеокодеков, часто используется для потокового видео и Blu-ray
- **H.265 (HEVC)** — Более эффективный видеокодек, обеспечивающий лучшее качество при меньшем размере файла по сравнению с H.264
- **MPEG-4**: Широко используемый стандарт для хранения видео и аудио данных
- **DivX**: Кодек, обеспечивающий хорошее качество видео при разумном размере файла
- **Xvid**: Открытая альтернатива DivX с высокой степенью сжатия и хорошим качеством изображения
