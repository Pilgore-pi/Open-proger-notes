■ С-стиль

Синтаксис:
```cs
(<type>) <expression>
```
Попытка привести выражение к заданному типу. Такой подход может генерировать исключение.

■ Через ключевое слово as

Синтаксис:

```cs
<expression> as <ref_type>
```

Ключевое слово «as» работает только с ссылочными типами. Оператор возвращает приведенный объект или «null», если приведение типов не удалось.

Если в цикле «foreach» явно указан тип переменной, то компилятор будет пытаться приводить элементы коллекции к указанному типу:

```cs
double[] nums = {1, 3, 5, 7};

foreach(int x in nums)
    Console.WriteLine(x);
```

## Ключевое слово is

Синтаксис:

```cs
<expression> is <type>
```

Если приведение типов возможно оператор возвращает true, в противном случае возвращает «false». Тип к которому приводится выражение может быть любым (класс, структура, делегат).

▽ Все преобразования происходят в “Runtime” (во время работы программы), так как проверка во время компиляции не всегда возможна.

## ▣ Упаковка и распаковка

Упаковка – это приведение значимого типа к object или к интерфейсу, который реализует данный значимый тип. При этом переменная копируется в специальный объект, который помещается в «кучу». Упаковка является неявной операцией.

Распаковка – это явное извлечение значимого типа из ссылочного объекта

```cs
int number = 228;

object o = number; //упаковка (boxing)

int i = (int)o; //распаковка (unboxing)

...

object o = 700u;

//int i = (int)o; – ошибка

int i = (int)(uint)o; //нет ошибки
```

Операции упаковки и распаковки являются дорогими операциями с точки зрения вычислений.

Упаковка может неявно происходить в следующих случаях:

При вызове метода у значимого типа (`5.ToString()`)
