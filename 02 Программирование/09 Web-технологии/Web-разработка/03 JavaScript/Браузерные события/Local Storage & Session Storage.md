Оба типа хранилища находятся на локальном компьютере пользователя. Большинство современных браузеров позволяют выделять от 5 Мб памяти (изменяется в настройках).

Хранилища привязаны к источнику (домен/протокол/порт). Это значит, что разные протоколы или поддомены определяют разные объекты хранилища, и они не могут получить доступ к данным друг друга.

Объекты хранилища `localStorage` и `sessionStorage` предоставляют одинаковые методы и свойства:

- `setItem(key, value)` – сохранить пару ключ/значение.
- `getItem(key)` – получить данные по ключу `key`.
- `removeItem(key)` – удалить данные с ключом `key`.
- `clear()` – удалить всё.
- `key(index)` – получить ключ на заданной позиции.
- `length` – количество элементов в хранилище.

Обращение к localStorage:
```js
window.localStorage
// or
localStorage
```

* Естественно можно обращаться свойствам хранилищ напрямую, но этого делать не рекомендуется, так как при модификации данных происходит событие `storage` и юзерю..

Перебор данных по ключам хранилища в JS:

```js
let keys = Object.keys(localStorage);
for(let key of keys) {
  alert(`${key}: ${localStorage.getItem(key)}`);
}
```

>Ключи и значения в хранилищах должны быть строками

# Local Storage

В браузерах определено хранилище Local Storage. В JS он представлен объектом localStorage.

Этот объект хранит пары `key-value`

* Данные не удаляются при закрытии браузера и перезагрузки ОС
* Данные Local Storage автоматически не удаляются



# Session Storage

Данные храняться, пока браузер не закрыт. В остальном полностью аналогичен объекту localStorage.
