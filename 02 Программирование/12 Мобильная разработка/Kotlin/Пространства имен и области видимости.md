В Kotlin понятия пространства имён и области видимости работают немного иначе, чем, например, в PHP или Python. В Kotlin нет ключевого слова `namespace`, как в PHP, но есть пакеты (packages), которые выполняют схожую роль.[3][7]

### Пространства имён (пакеты)

В Kotlin пространства имён реализуются через пакеты. Каждый файл Kotlin начинается с объявления пакета:
```kotlin
package com.example.myapp
```
Это позволяет избежать конфликтов имён между классами, функциями и другими объявлениями. Если два класса имеют одинаковые имена, но находятся в разных пакетах, они не конфликтуют.[3]

Для использования классов из других пакетов применяется ключевое слово `import`:
```kotlin
import com.example.otherpackage.MyClass
```
Это позволяет обращаться к классу по короткому имени внутри текущего файла.[3]

### Области видимости

Область видимости в Kotlin определяет, где переменные, функции и классы доступны. В Kotlin есть несколько модификаторов доступа:
- `public` — доступен отовсюду (по умолчанию для верхнеуровневых объявлений).
- `private` — доступен только внутри файла или класса.
- `protected` — доступен внутри класса и его наследников.
- `internal` — доступен только внутри модуля.[7][3]

Кроме того, Kotlin предоставляет функции области видимости (scope functions): `let`, `run`, `with`, `apply`, `also`. Эти функции позволяют временно изменить контекст выполнения кода и упростить работу с объектами:
```kotlin
val person = Person("Alice", 20)
person.apply {
    moveTo("London")
    incrementAge()
}
```
Внутри блока `apply` объект доступен без явного указания имени, что делает код более читаемым.[7][3]

### Примеры

- Пакеты помогают организовать код и избежать конфликтов имён.
- Модификаторы доступа контролируют, где можно использовать объявления.
- Функции области видимости упрощают работу с объектами и делают код более лаконичным.
