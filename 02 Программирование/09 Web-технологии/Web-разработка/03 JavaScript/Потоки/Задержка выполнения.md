Существуют встроенные функции отложенного выполнения кода:

1. `setTimeout(func, delay)` — однократное выполнение через указанное время

2. `setInterval(func, delay)` — постоянное выполнение через интервалы времени

3. `clearTimeout(timerId)` — отменяет отложенное выполнение кода

4. `clearInterval(timerId)` — прерывает интервальное выполнение кода

Каждый вызов `setTimeout() & setInterval()` возвращает идентификатор таймера, который используется для при выполнении кода. Так как таймеров может быть много, для остановки определенного таймера нужно указать его идентификатор.

>Во время показа `alert` время тоже идёт в большинстве браузеров

>Когда функция передаётся в `setInterval/setTimeout`, на неё создаётся внутренняя ссылка и сохраняется в планировщике. Это предотвращает попадание функции в сборщик мусора, даже если на неё нет других ссылок.

Можно выставить нулевую задержку выполнения (задержка = 0 или не указана). В таком случае код будет выполняться снова и снова так быстро, как это возможно.
