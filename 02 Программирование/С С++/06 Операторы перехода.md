## Оператор break

> Оператор `break` выходит из ближайшего цикла (`for`, `while`, `do-while`) или заканчивает выполнение блока `case` внутри `switch`.

```cpp
switch (choice) {
    case 1: std::cout << a + b << std::endl; break; // завершение блока case
    case 2: // отсутствует break, переход к case 3
    case 3: std::cout << "Неподдерживаемая операция" << std::endl; break;
    default: std::cout << 0 << std::endl; break;
}

for (int i = 0; i < 10; i++) {
    std::string choice;
    std::getline(std::cin, choice);
    if (choice == "5") break; // прерывание цикла

    std::cout << "your choice is " << choice << std::endl;
}
```

### Особенности в C/C++

- В C/C++ нет поддержки меток для выхода из внешних циклов через `break` — `break` всегда прекращает ближайший цикл или `switch`.
- Прерывание вложенных циклов по метке делается через `goto`.

```cpp
for (int i = 0; i < 3; i++) {
    std::cout << "Внешний цикл i = " << i << std::endl;

    for (int j = 0; j < 5; j++) {
        if (j == 2)
            break; // прервать только внутренний цикл
        std::cout << "  Внутренний цикл j = " << j << std::endl;
    }
}

std::cout << "Циклы завершены" << std::endl;
```

## Оператор continue

> `continue` пропускает оставшуюся часть текущей итерации цикла и переходит к следующей итерации ближайшего цикла (`for`, `while`, `do-while`). Не используется в `switch`.

```cpp
for (int i = 0; i < 3; i++)
    for (int j = 0; j < 5; j++) {
        if (j == 3)
            continue; // пропустить оставшийся код и перейти к следующей итерации цикла j

        for (int k = 0; k < 5; k++) {
            std::cout << "i=" << i << ", j=" << j << ", k=" << k << std::endl;
        }
    }
```

## Оператор return

> Завершает выполнение функции и возвращает управление вызывающему коду, при необходимости со значением.

```cpp
int GetSum(int n1, int n2) {
    return n1 + n2;
}

void Print(const std::string& str) {
    if (str.empty())
        return; // выход из функции

    std::cout << str << std::endl;
}
```

## Оператор throw (C++ исключения)

> `throw` выбрасывает исключение. Поток выполнения прерывается, управление передается ближайшему обработчику (`catch`).

```cpp
throw std::runtime_error("сообщение об ошибке");

try {
    int x = 5 / 0; // в C++ это не выбросит исключение, но для примера
} catch (const std::exception& ex) {
    std::cout << "Ошибка: " << ex.what() << std::endl;
    throw; // повторное выбрасывание исключения
}
```

- В отличие от C#, в C++ деление на ноль для int не вызывает исключение автоматически.
- Исключения создаются и обрабатываются с помощью классов и конструкций `try-catch`.

## Оператор goto

> Передает управление по метке внутри текущей функции. Используется для перехода в пределах одной функции, не рекомендуется к использованию без необходимости.

```cpp
int i = 0;

start:
std::cout << i << std::endl;
i++;

if (i < 3)
    goto start;

// Альтернативой является цикл:
for (int i = 0; i < 3; i++) {
    std::cout << i << std::endl;
}
```

Пример выхода из вложенных циклов через `goto`:

```cpp
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 1)
            goto end; // выход из обоих циклов
        std::cout << i << ", " << j << std::endl;
    }
}

end:
std::cout << "Вышел из циклов" << std::endl;
```

### Отличия в использовании

| Особенность        | C#                                                                | C/C++                                                         |
| :----------------- | :---------------------------------------------------------------- | :------------------------------------------------------------ |
| Метки с break      | Поддерживаются (выход по метке из внешнего цикла)                 | Нет, для выхода из внешних циклов используется `goto`         |
| Использование goto | Рекомендуется избегать, поддерживает `goto case` и `goto default` | Используется внутри функций, без ограничений на имена меток   |
| Исключения (throw) | Генерация и переброс исключений с встроенной системой обработки   | Исключения реализованы как объекты и обрабатываются try-catch |
| continue в switch  | Не используется                                                   | Не используется                                               |
