> **Частичные классы** — это классы, которые определяются в разных файлах

Функционально частичный класс не отличается от обычного класса. Обычно, частичные классы используются совместно с генераторами кода. К примеру, программист обеспечивает логику формы WinForms, а генератор кода, создает остальной код для настройки зависимостей.

Другим сценарием использования может послужить совместная разработка одного класса. Каждый программист может разрабатывать свою часть класса

```csharp
// file MyClassPart1.cs
internal partial class MyClass {
    
}

// file MyClassPart2.cs
internal partial class MyClass {
    
}
```

**Наследование**. Обязательно ли указывать базовый класс в каждой части определения?
