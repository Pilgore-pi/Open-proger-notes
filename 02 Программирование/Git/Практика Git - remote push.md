Здесь будет рассмотрен процесс выгрузки хранилища Obsidian на удаленный репозиторий GitHub. Данный процесс будет состоять из следующих шагов:

1. Регистрация на GitHub
2. Создание облачного репозитория
3. Установка Git
4. Создание локальной версии репозитория
5. Добавление нужных файлов в индекс для последующего коммита
6. Коммит
7. Отправка изменений на удаленный репозиторий

### 1 Регистрация на GitHub

Регистрация довольно тревиальна и не требует подробных разъяснений

### 2 Создание облачного репозитория

В панели управления сайта, рядом с иконкой профиля есть кнопка создания нового репозитория. Далее создае новый репозиторий и выбираем нужные нам параметры (например публичность / приватность репозитория)

### 3 Создание локального репозитория

- Устанавливаем Git
- Открываем консоль Git Bash
- Задаем пользователя, от имени которого будут выполняться действия:

```git
git config --global user.name "Hero"
git config --global user.email "hero@gmail.com"

// Можно получить данные текущего пользователя с помощью этих команд
git config --global --list

// Свойства пользователя по-отдельности
git config --get user.name
git config --get user.email
```

- Прописываем команду перехода в каталог, который будет являться корневым для нашего хранилища Obsidian

```git
cd "C:\Конспекты\Мои хранилища\ProgerNotes"
```

- Инициализируем в этом каталоге репозиторий Git, с помощью команды:

```git
git init
```

Это создаст скрытый каталог `.git` с метаданными репозитория внутри указанного каталога `ProgerNotes`

- Перед тем, как добавить нужные файлы, создадим файл `.gitignore` в корне хранилища Obsidian и укажем каталоги и файлы, которые Git должен игнорировать при добавлении в индекс и коммитах ([подробнее о .gitignore](https://git-scm.com/docs/gitignore)):

```gitignore
*.mp4
02 Программирование\01 .NET\C# 7.pdf
02 Программирование\01 .NET\Конспекты в тетради
```

- Добавляем нужные файлы в новосозданный репозиторий (в текущем примере добавляются все файлы). Нужно учитывать, что есть ограничение по объему загружаемых файлов (один файл не должен быть больше 100 МБ)

```git
git add --all
```

- Делаем `commit`

```git
git commit -m "первый коммит"
```
