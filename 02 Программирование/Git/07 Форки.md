
> **Форк (fork)** — это точная копия существующего репозитория, созданная на каком-либо облачном хостинге.

В GitHub форк синхронизируется с оригиналом и наследует уровень доступа оригинала (`private`, `public`).

## Зачем нужен форк?

- Позволяет работать с чужим проектом, внося изменения, не затрагивая оригинальный репозиторий

- Используется для экспериментов, исправления ошибок, добавления новых функций

- После внесения изменений можно отправить **pull request** — запрос на включение изменений в оригинальном репозиторий. Если авторы оригинала одобрят, они примут изменения и сольют их с основным проектом

- Форк помогает поддерживать идеологию открытого исходного кода (Open Source), позволяя свободно копировать и улучшать проекты

## Отличия веток, клонов и форков


| Параметр           | Форк (Fork)                                              | Клон (Clone)                            | Ветка (Branch) |
| ------------------ | -------------------------------------------------------- | --------------------------------------- | -------------- |
| Где создаётся      | На GitHub в вашем аккаунте                               | Локально на вашем компьютере            |                |
| Связь с оригиналом | Сохраняется, видна на GitHub                             | Нет прямой связи, это просто копия      |                |
| Доступ             | Есть полный доступ в форк                                | Нет доступа к оригинальному репозиторию |                |
| Цель               | Внесение изменений с возможностью отправить pull request | Локальная работа, запуск, тестирование  |                |
