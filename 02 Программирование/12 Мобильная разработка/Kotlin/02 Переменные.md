
## Базовые типы данных

### Числа

**Целые числа:**

| Тип данных | Диапазон        | Размер (байт) |
| ---------- | --------------- | ------------- |
| Byte       | `[-128, 127]`   | 1             |
| Short      | `[-2^15, 2^15)` | 2             |
| Int        | `[-2^31, 2^31)` | 4             |
| Long       | `[-2^63, 2^63)` | 8             |

В Kotlin отсутствуют беззнаковые целочисленные типы в классическом понимании. Вместо них используются специальные типы с префиксом `U`: `UByte`, `UShort`, `UInt`, `ULong`. Однако они реализованы как встроенные классы, а не как примитивные типы.

**Числа с плавающей точкой:**

| Тип данных | Диапазон                                                 | Размер (байт) | Точность     |
| ---------- | -------------------------------------------------------- | ------------- | ------------ |
| Float      | от $±1.4\ \times\ 10^{−45}$ до $±3.4\ \times\ 10^{38}$  | 4             | 7 знаков     |
| Double     | от $±4.9\ \times\ 10^{−324}$ до $±1.7\ \times\ 10^{308}$ | 8             | 15-16 знаков |

Точность чисел с плавающей точкой зависит от того, насколько большое число. Если число очень большое, то в память в приоритете выделяется для больших разрядов, а дробная часть сокращается, поэтому такие числа называются "числа с плавающей точкой"

**Числа с произвольной точностью:**

| Тип данных | Описание                                | Размер (байт) |
| ---------- | --------------------------------------- | ------------- |
| BigInteger | Целые числа произвольной длины          | переменный    |
| BigDecimal | Дробные числа с произвольной точностью | переменный    |

Типы `BigInteger` и `BigDecimal` предназначены для работы с числами, которые не помещаются в стандартные типы, или когда требуется произвольная точность вычислений. Они находятся в пакете `java.math`

### Дата и время

Kotlin не имеет встроенных типов для работы с датой и временем. Вместо этого используются классы из Java:

| Тип данных | Описание |
| ---------- | -------- |
| LocalDate  | **Дата без времени**; представляет дату в формате год-месяц-день |
| LocalTime  | **Время**; представляет время суток в формате часы-минуты-секунды-наносекунды |
| LocalDateTime | **Дата и время**; комбинация даты и времени без информации о часовом поясе |
| ZonedDateTime | **Дата, время и часовой пояс**; полная информация о моменте времени с учетом часового пояса |

Эти типы находятся в пакете `java.time`

## Объявление переменных

В Kotlin существует два ключевых слова для объявления переменных:

### Изменяемые переменные (`var`)

```kotlin
var name = "John"
var age = 25
var height = 1.75
var isStudent = true
```

Переменные, объявленные с помощью `var`, могут изменять свое значение после инициализации. Тип данных переменной определяется автоматически на основе присвоенного значения (тип-вывод).

### Неизменяемые переменные (`val`)

```kotlin
val PI = 3.14159
val MAX_USERS = 100
val DEFAULT_NAME = "Guest"
```

Переменные, объявленные с помощью `val`, не могут изменять свое значение после инициализации. Они эквивалентны константам в других языках, но в отличие от C#, могут быть инициализированы в момент объявления или позже (один раз).

### Явное указание типа данных

```kotlin
var count: Int = 10
val message: String = "Hello, Kotlin!"
var temperature: Double = 36.6
val isActive: Boolean = true
```

Явное указание типа данных полезно для повышения читаемости кода и предотвращения ошибок типизации.

### Отложенная инициализация

```kotlin
var userName: String
userName = "Alice"

val createdDate: String
createdDate = "2025-01-15"
```

В Kotlin переменные могут быть объявлены без инициализации, но в таком случае необходимо явно указать тип данных. Переменная должна быть инициализирована перед первым использованием.

## Именование переменных

> **Идентификатор переменной** — это уникальное имя переменной, по которому можно к ней обращаться

**Правила именования:**

- Имя переменной (идентификатор) не должно начинаться с цифры
- Имя переменной может содержать любые буквенные символы кодировки UTF-16, цифры и символ `_`
- Имя переменной не должно совпадать с зарезервированным оператором языка
- Имя переменной может содержать спец. символы (например, `$`), но это не рекомендуется

```kotlin
var myVariable = 10
var my_variable = 20
var MY_VARIABLE = 30
var моя_переменная = 40 // не рекомендуется
var _myVariable = 50
var `when` = 60 // Использование обратных кавычек для зарезервированных слов
```

Символ **`` ` ``** (обратная кавычка) нужен для того, чтобы использовать зарезервированные слова языка Kotlin в качестве имен переменных. Этот символ позволяет избежать конфликтов с операторами языка.

Недопустимые идентификаторы:

```kotlin
var 111var = 10
var @var = 20
var 6var = 30
var myVariable@ = 40
```

### Виды нотаций

| Нотация           | Пример           |
| ----------------- | ---------------- |
| верхняя верблюжья | UpperCamelCase   |
| нижняя верблюжья  | lowerCamelCase   |
| змеиная           | snake_case       |
| толстая змеиная   | FAT_SNAKE_CASE   |
| шашлычная         | kebab-case       |

В Kotlin рекомендуется использовать **нижнюю верблюжью нотацию** (`lowerCamelCase`) для имен переменных и функций, а **верхнюю верблюжью нотацию** (`UpperCamelCase`) для имен классов и объектов

> К любому литералу применима точка (.), то есть существует возможность обратиться к методам и свойствам неименованного объекта

#### Escape-последовательности

> **Escape-последовательности** — это специальные подстроки, которые воспринимаются как специальные символы, которые было бы неудобно вводить вручную. Escape-последовательности нужны, чтобы вставлять знаки табуляции, перевода на новую строку или вставлять специальные символы, которых нет на клавиатуре

| Escape-последовательность | Описание |
| ------------------------- | -------- |
| `\\`                      | Экранированный обратный слеш (`'\'`) |
| `\0`                      | Нулевой символ  (символ UTF-16 с кодом `0`) |
| `\t`                      | Символ табуляции                            |
| `\n`                      | Символ перевода строки                      |
| `\r`                      | Символ возврата каретки (устаревший символ, используемый во времена печатных машинок) |
| `\'`                      | Одинарная кавычка                           |
| `\"`                      | Двойная кавычка                             |
| `\$`                      | Символ доллара (используется для экранирования в интерполяции строк) |
| `\uXXXX`                  | Символ с кодом XXXX в кодировке UTF-16 (например, `\u0411` — буква `Б`) |

#### Интерполяция строк

```kotlin
val name = "Tom"
val age = 34
val height = 1.7

// Подстановка по выражениям
println("Имя: $name Возраст: $age Рост: ${height}м")

// Подстановка с выражениями
println("10 + 33 = ${10 + 33}")
// 10 + 33 = 43

// Подстановка объектов
val person = "Alice"
println("Привет, $person!")
```

Когда в строковом литерале используется символ `$`, то после него можно указать имя переменной или выражение в фигурных скобках. Выражение вычисляется и его результат подставляется в строку.

Если в интерполируемой строке нужно записать символ доллара, то он должен быть экранирован:

```kotlin
println("Цена: \$100")
// Цена: $100
```

#### Многострочные строки (raw strings)

Многострочные строки в Kotlin игнорируют escape-последовательности (кроме `\$` и `\\`), но при этом к ним можно применять интерполяцию:

```kotlin
// обычная строка
val mystr1 = "c:\\myfolder\\myfile.txt"

// Многострочная строка (raw string)
val mystr2 = """c:\myfolder\myfile.txt"""

val fileName = "myfile.txt"

// Многострочная строка с интерполяцией
val mystr3 = """c:\myfolder\$fileName"""

// Многострочная строка с отступом
val xml = """
    <element attr="content">
        <body>
        </body>
    </element>
""".trimIndent()
```

Многострочные строки всегда начинаются и заканчиваются тройными двойными кавычками. Они сохраняют все символы, включая переводы строк и пробелы. Для удаления ведущих пробелов можно использовать функцию `trimIndent()`.

### Ссылочные литералы

```kotlin
null                                    // нулевая ссылка
LocalDate.of(2025, 1, 1)               // 2025-01-01
LocalDateTime.of(2025, 1, 1, 8, 30, 0) // 2025-01-01T08:30:00
String(charArrayOf('a', 'b'))          // новая строка на основе массива символов
```

#### Литерал `null`

Литерал `null` представляет отсутствие значения. В Kotlin система типов различает типы, которые могут быть `null` (nullable types) и типы, которые не могут быть `null` (non-null types):

```kotlin
var name: String = "John"      // не может быть null
var age: Int? = null           // может быть null

var city: String?
city = "Moscow"
city = null                    // допустимо
```

Тип, который может быть `null`, обозначается символом `?` после имени типа.

## Константы

> **Константа** — это именованная переменная, значение которой известно на этапе компиляции и не может быть изменено. Константы не изменяемы никаким образом

В Kotlin константы объявляются с помощью ключевого слова `const val`. Они должны быть объявлены на уровне файла или внутри объекта (object), но не внутри функций.

```kotlin
// Объявление констант на уровне файла
const val PI = 3.14159
const val MAX_USERS = 100
const val DEFAULT_NAME = "Guest"
const val DEFAULT_SYMBOL = 'o'

// Инициализация другой константой
const val RADIUS = 5
const val DIAMETER = RADIUS * 2

// Инициализация константным выражением
const val AREA = PI * RADIUS * RADIUS
```

Константы внутри класса или объекта:

```kotlin
object Config {
    const val API_URL = "https://api.example.com"
    const val TIMEOUT = 30000L
}

class Settings {
    companion object {
        const val VERSION = "1.0.0"
        const val DEBUG_MODE = false
    }
}
```

Отличие от `val`:

```kotlin
val userName = "Alice"      // переменная, не может быть изменена, но вычисляется во время выполнения
const val MAX_AGE = 120     // константа, вычисляется на этапе компиляции
```

**Ограничения для констант:**

- Константы могут быть только примитивных типов (`Int`, `Long`, `Float`, `Double`, `Boolean`, `Char`) и типа `String`
- Константы должны быть инициализированы литералами или выражениями, которые можно вычислить на этапе компиляции
- Константы должны быть объявлены на уровне файла или внутри объекта/класса (в блоке `companion object`)
- Константы не могут быть объявлены внутри функций

При компиляции программы, все места, где была упомянута константа заменяются на значение этой константы.
