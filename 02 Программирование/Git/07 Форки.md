
> **Форк (fork)** — это точная копия существующего репозитория, созданная на каком-либо облачном хостинге.

В GitHub форк синхронизируется с оригиналом и наследует уровень доступа оригинала (`private`, `public`).

## Зачем нужен форк?

- Позволяет работать с чужим проектом, внося изменения, не затрагивая оригинальный репозиторий

- Используется для экспериментов, исправления ошибок, добавления новых функций

- После внесения изменений можно отправить **pull request** — запрос на включение изменений в оригинальном репозиторий. Если авторы оригинала одобрят, они примут изменения и сольют их с основным проектом

- Форк помогает поддерживать идеологию открытого исходного кода (Open Source), позволяя свободно копировать и улучшать проекты

## Отличия веток, клонов и форков

| Характеристика      | Ветка (Branch)                                                      | Форк (Fork)                                                         | Клон (Clone)                              |
| :------------------ | :------------------------------------------------------------------ | :------------------------------------------------------------------ | :---------------------------------------- |
| Место создания      | Внутри одного репозитория                                           | Отдельный репозиторий на облачном хостинге, на аккаунте             | Локальная копия репозитория на компьютере |
| Уровень             | Локальный (в рамках репозитория)                                    | Уровень хостинга (GitHub)                                           | Локальный                                 |
| Контроль            | Общий с владельцем репозитория                                      | Полный контроль у форкера                                           | Полный контроль на локальной машине       |
| Связь с оригиналом  | Общая история коммитов внутри репозитория                           | Сохраняется связь на GitHub, но репозиторий независим               | Нет прямой связи, просто копия            |
| Цель                | Параллельная разработка внутри проекта                              | Независимая работа с чужим проектом, внесение изменений             | Локальная работа, тестирование, запуск    |
| Необходимые права   | Требуются права на запись в репозиторий                             | Не требуются права на запись в оригинал                             | Не требуются права                        |
| Использование       | Для работы над новыми функциями, исправлениями в рамках репозитория | Для внесения изменений в чужой проект или самостоятельного развития | Для локальной работы с кодом              |
| Влияние на оригинал | Влияет при слиянии веток (merge)                                    | Не влияет до создания pull request                                  | Не влияет                                 |
