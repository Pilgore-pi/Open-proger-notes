
По всему миру компьютеры связаны между собой сетями (локальными, глобальными, Интернетом). Все подключенные устройства (***хосты***) связаны каналами связи — кабели Ethernet, Wi-Fi, маршрутизаторами и др.

> **Маршрутизатор** — устройство, которое объединяет компьютеры в подсети и контролирует передачу данными между ними по определенным правилам. Пример маршрутизатора — роутер.

В зависимости от охвата территории компьютерные сети бывают:

1. **Персональные** — Personal Area Network (PAN)
2. **Локальные** — Local Area Network (LAN)
3. **Городские** — Metropolitan Area Network (MAN)
4. **Глобальные** — Wide Area Network (WAN)

> **Протоколы** — это соглашения о том, как пакеты будут передаваться по каналам коммуникации. Таким образом протоколы упорядочивают и стандартизируют взаимодействие хостов.

Семейство протоколов `TCP/IP` используются для передачи данных по сети. Основные протоколы этого семейства:

1. Internet Protocol (IP)
2. Transmission Control Protocol (TCP)
3. User Datagram Protocol (UDP)
4. Domain Network System (DNS)
5. Dynamic Host Configuration Protocol (DHCP)
6. Internet Control Message Protocol (ICMP)

IP представляет сетевой уровень. Он использует нижележащие уровни, которые представляют физические каналы коммуникации — кабели Ethernet и т.д., для передачи пакетов с данными другому хосту.

Выше IP располагается *транспортный уровень*, который образуют протоколы TCP и UDP. Эти протоколы используют определенные порты для передачи данных. TCP позволяет отследить потерю пакетов и их дублирование при передаче. UDP подобного не позволяет сделать и нацелен на простую передачу данных.

Однако приложение взаимодействует с уровнем TCP / UDP не напрямую, а через специальный API, который предоставляют сокеты.

>**Сокеты** — это интерфейс доступа к определенному порту определенного хоста. Он предназначен для создания сетевых приложений и опирается на встроенные возможности операционной системы.

В зависимости от используемого протокола различают два вида сокетов:

1. Потоковые (stream socket). Используется протокол TCP
2. Дейтаграммные сокеты (datagram socket). Используется протокол UDP

Для идентификации хостов в сети используют адреса. Существует несколько различных протоколов адресов. Самый распространенный — `IPv4`, по стандарту которого адрес представлен как 4 однобайтных числа. Такой адрес называется IP адресом.

> **IP-адрес** – это уникальный адрес, идентифицирующий устройство в сети

Для удобства IP адреса сайтов были заменены на **домены** (доменные имена, адреса), например `"www.microsoft.com"`. Для протокола IP доменных имен не существует.

Кроме адреса при сетевых взаимодействиях используются ***порты***.

>Порт представляет собой неотрицательное 2-байтное число. По сути порт — это виртуальный разъем, как порт USB на материнской плате, через который происходит обмен данными с системой и другими программами.

Порты существуют для разграничения данных между приложениями. Один порт может прослушивать толко одно приложение (уточнить как, типа один процесс, поток или что?)

Ключевыми компонентами сетевого взаимодействия являются клиент и сервер. Клиент посылает запрос, а сервер получает его, обрабатывает и посылает обратно клиенту некоторый ответ. Простейший пример — веб-браузер, который служит в качестве клиента, отправляя запрос на сайт. А сайт выступает в качестве сервера, отправляя браузеру ответ
