
## Создание обратного коммита с помощью `git revert`

- Если коммит уже опубликован и нельзя переписывать историю, лучше использовать `git revert`. Эта команда создаёт новый коммит, который отменяет изменения выбранного коммита, сохраняя историю.

Пример:
```bash
git revert HEAD
```

- Эта команда безопасна для общих веток, так как не меняет историю, а лишь добавляет новый коммит с отменой изменений

## Откат к предыдущему состоянию ветки

- Команда `git reset` перемещает указатель текущей ветки (HEAD) на указанный коммит. При этом есть несколько режимов:
    
    - `git reset --soft HEAD~1` — откат на один коммит назад, при этом изменения из отменённого коммита остаются в индексе (staged), можно сразу сделать новый коммит.
        
    - `git reset --mixed HEAD~1` (по умолчанию) — откат на один коммит назад, изменения остаются в рабочей директории, но не проиндексированы.
        
    - `git reset --hard HEAD~1` — откат на один коммит назад с удалением всех изменений из рабочего каталога и индекса, т.е. состояние репозитория полностью возвращается к предыдущему коммиту.

## Временный просмотр предыдущей версии

- Если нужно просто посмотреть состояние репозитория на предыдущем коммите, можно переключиться на него командой: `git checkout <commit-hash>`

- Это переводит репозиторий в состояние "`detached HEAD`", где вы можете просмотреть файлы, но не изменять основную ветку без дополнительных действий
