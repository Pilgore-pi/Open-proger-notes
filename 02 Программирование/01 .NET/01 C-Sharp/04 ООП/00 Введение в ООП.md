
> Объектно-ориентированное программирование (ООП) — это парадигма, ориентированная на работу с объектами и их свойствами

* **Парадигма программирования** — совокупность идей и понятий, определяющих стиль написания компьютерных программ

ООП является самой популярной парадигмой программирования и широко используется повсеместно. ООП имеет следующие преимущества:

- Масшабируемость
- Гибкость
- Концепция ООП основана на понятиях из реального мира

Недостатки:

- Сложность в понимании и изучении. Конкретные механизмы и особенности ООП могут быть запутанными и весьма сложными как в плане понимания, так и в плане логики.лоз9гпнапмнпол
- Время разработки является следствием из сложности парадигмы. Для маленьких проектов использование объектно-ориентированного кода может быть не оправдано и излишне усложнять программу

> Ключевыми понятиями ООП являются **объект** (отражает объект реального мира) и **класс** (шаблон этого объекта)

**Пример**. Существует класс **`Детский сад`**, который описывает чертежи здания и допустимую занимаемую площадь. Конкретные садики "Казачок" и "Дельфинчик" имеют свои чертежи и разную занимаемую площадь, то есть имеют различные характеристики, но сам набор этих характеристик у них одинаковый. Если у "Казачка" есть характеристика "высота забора", то и у "Дельфинчика" она есть, потому что они описаны одним и тем же шаблоном "Детский сад". 

## Другие парадигмы

- **Процедурная**. Последовательность команд, изменяющих состояние программы

- **Функциональная**. Неизменяемость, чистые функции, функции высшего порядка. В .NET: LINQ, F#, делегаты, лямбды

- **ООП**. Последовательность команд, изменяющих состояние программы

- **Декларативная** описываешь ЧТО нужно получить, а не КАК это сделать. В C#: `var adults = people.Where(p => p.Age >= 18); // LINQ`. XAML, SQL

- **Логическая**. Prolog. программа = набор фактов и правил логического вывода. Система сама находит решение через унификацию и бэктрекинг

```prolog
// Пример на Prolog (в .NET нет нативной поддержки)
parent(tom, bob).
parent(tom, liz).
sibling(X, Y) :- parent(Z, X), parent(Z, Y), X \= Y.
```

- **Реактивная**

- **Конкатенативная**. программа = композиция функций, работающих со стеком

```forth
// Пример на Forth (в .NET нет прямой поддержки)
5 3 + 2 *  // (5+3)*2 = 16
```

- **Потоковая**. программа = граф узлов, обрабатывающих потоки данных

```csharp
// TPL Dataflow в .NET
var multiplyBlock = new TransformBlock<int, int>(n => n * 2);
var printBlock = new ActionBlock<int>(n => Console.WriteLine(n));

multiplyBlock.LinkTo(printBlock);
multiplyBlock.Post(5); // Выведет 10
```

- **Событийно-ориентированная**. поток управления определяется событиями

```csharp
button.Click += (sender, e) => {
    MessageBox.Show("Clicked!");
};
```

В .NET: события, делегаты, EventAggregator паттерн

- **Аспектно-ориентированная**. отделение сквозной функциональности (логирование, транзакции)

```csharp
// PostSharp
[Log]
public void ProcessOrder(Order order) {
    // Логирование добавляется автоматически
}
```

- **Агентно-ориентированная**. независимые агенты с собственным состоянием, взаимодействующие через сообщения

```csharp
// Akka.NET пример
public class GreetingActor : ReceiveActor {
    public GreetingActor() {
        Receive<Greet>(greet => {
            Console.WriteLine($"Hello {greet.Who}");
        });
    }
}
```

- **Прототипная**. объекты клонируются из других объектов, нет классов. JavaScript

- **Вероятностная**. Программы работают с вероятностными распределениями. В .NET можно использовать Infer.NET от Microsoft Research

- **Квантовая**. Программирование квантовых компьютеров. Q# от Microsoft — это язык для квантовых вычислений в .NET экосистеме

```qsharp
operation SampleQuantumRandomNumberGenerator() : Result {
    use q = Qubit();
    H(q);
    return M(q);
}
```
