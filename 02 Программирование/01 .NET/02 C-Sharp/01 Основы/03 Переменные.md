
## Базовые типы данных

### Числа

**Целые числа:**

| Тип данных | Диапазон        | Размер (байт) |
| ---------- | --------------- | ------------- |
| byte       | `[0, 255]`      | 1             |
| sbyte      | `[-128, 127]`   | 1             |
| short      | `[-2^15, 2^15)` | 2             |
| ushort     | `[0, 2^16)`     | 2             |
| int        | `[-2^31, 2^31)` | 4             |
| uint       | `[0, 2^32)`     | 4             |
| long       | `[-2^63, 2^63)` | 8             |
| ulong      | `[0, 2^64)`     | 8             |

Целочисленные типы, которые имеют префикс `u` хранят только неотрицательные значения (`u` обозначает `unsigned` — без знака). В случае с `sbyte` `s` означает `signed`, то есть "со знаком"

**Числа с плавающей точкой:**

| Тип данных | Диапазон                                                 | Размер (байт) | Точность     |
| ---------- | -------------------------------------------------------- | ------------- | ------------ |
| float      | от $±1.5\ \times \ 10^{−45}$ до $±3.4\ \times\ 10^{38}$  | 4             | 7 знаков     |
| double     | от $±5.0\ \times\ 10^{−324}$ до $±1.7\ \times\ 10^{308}$ | 8             | 15-16 знаков |

Точность чисел с плавающей точкой зависит от того, насколько большое число. Если число очень большое, то в память в приоритете выделяется для больших разрядов, а дробная часть сокращается, поэтому такие числа называются "числа с плавающей точкой"

**Числа с фиксированной точкой:**

| Тип данных | Описание                                               | Размер (байт) | Точность     |
| ---------- | ------------------------------------------------------ | ------------- | ------------ |
| decimal    | от $±1.0\ \times\ 10^{−28}$ до $±7.9\ \times\ 10^{28}$ | 16            | 28-29 знаков |

Числа с фиксированной точкой всегда имеют одинаковую точность дробной части, независимо от размера числа.

Тип `decimal` предназначен для задач, где критично важна большая точность вычислений

## Именование переменных


> **Идентификатор переменной** — это уникальное имя переменной, по которому можно к ней обращаться

**Правила именования:**

- Имя переменной (идентификатор) не должно начинаться с цифры или спец. символа
- Имя переменной может содержать любые буквенные символы кодировки UTF-16, цифры и символ `_`. Однако настоятельно рекомендуется использовать только ASCII символы (любые латинские буквы и цифры)
- Имя переменной не должно совпадать с зарезервированным оператором языка, либо необходимо применять спец. символ `@` перед именем переменной во избежание конфликтов

```csharp
int myVariable;
int my_variable;
int MY_VARIABLE;
int _my_variable;
int @my_variable; // Символ @ игнорируется
int \u005my_variable;
int @int; // Переменная с именем "int"
```

Символ **@** нужен для того, чтобы убрать неоднозначаность между оператором языка C\# и именем переменной. Этот символ позволяет называть переменные также, как и операторы языка.
В коде всегда нужно писать символ **@**, если переменная объявлена с именем, содержащим этот символ

Недопустимые идентификаторы:

```csharp
int 111var;
int @@var;
int @6var;
int true;
int i∈Z;
int myVariable@;
```

#### Escape-последовательности

Escape-последовательности — это специальные подстроки, которые воспринимаются как специальные символы, которые было бы неудобно вводить вручную. Escape-последовательности нужны, чтобы вставлять знаки табуляции, перевода на новую строку или вставлять специальные символы, которых нет на клавиатуре


| Escape-последовательность | Описание                                                                              |
| ------------------------- | ------------------------------------------------------------------------------------- |
| `\\`                      | Экранированный обратный слеш (`'\'`)                                                  |
| `\0`                      | Нулевой символ  (символ UTF-16 с кодом `0`)                                           |
| `\t`                      | Символ табуляции                                                                      |
| `\v`                      | Символ вертикальной табуляции                                                         |
| `\n`                      | Символ перевода строки                                                                |
| `\r`                      | Символ возврата каретки (устаревший символ, используемый во времена печатных машинок) |
| `\a`                      | Звуковой сигнал                                                                       |
| `\b`                      | Символ backspace                                                                      |
| `\x65` (`A` английская)   | символ ASCII с кодом 65                                                               |
| `\u0411` (`Б` русская)    | символ UTF-16 с кодом 411                                                             |

#### Интерполяция строк

```csharp
string name = "Tom";
int age = 34;
double height = 1.7;

// Подстановка по выражениям
Console.WriteLine($"Имя: {name} Возраст: {age} Рост: {height}м");

// Подстановка с порядком
Console.WriteLine("Имя: {0} Возраст: {2} Рост: {1}м", name, height, age);
```

Когда перед строковым литералом ставится `$`, то в фигурных скобках можно вставить любое выражение, не содержащее фигурных скобок

```csharp
Console.WriteLine($"10 + 33 = {10 + 33}");
// 10 + 33 = 43
```

Если в интерполируемой строке нужно записать фигурную скобку, то пишется двойная фигурная скобка:

```csharp
Console.WriteLine($"10 + 33 = {{ {10 + 33}, целое число }}");
// 10 + 33 = { 43, целое число }
```

#### Буквальные и необрабатываемые строки

Буквальные строки игнорируют любые escape-последовательности, но при этом к буквальным строкам можно применять интерполяцию.

```csharp
// обычная строка
string mystr1 = "c:\\myfolder\\myfile.txt";

// Буквальная строка (verbatim string)
string mystr2 = @"c:\myfolder\myfile.txt";

string fileName = "myfile.txt";

// Буквальная строка с интерполяцией
string mystr3 = $@"c:\myfolder\{fileName}";
```

Необрабатываемые строки (raw strings) позволяют не только игнорировать escape-последовательности, но и использовать двойные кавычки в тексте и переводы строк:

```csharp
var singleLine = """This is a "raw string literal". It can contain characters like \, ' and ".""";

var xml = """
        <element attr="content">
            <body>
            </body>
        </element>
        """;
```

В необрабатываемых строках также можно использовать интерполяцию

#### Литерал `default`

Литерал `default` — это контекстный литерал, который представляет значение по умолчанию для текущего типа данных. Для каждого типа данных `default` возвращает разные значения:


| Тип данных          | Значение `default`                                           |
| ------------------- | ------------------------------------------------------------ |
| Любой ссылочный тип | `null`                                                       |
| Любая структура     | Структура, у которой все свойства имеют стандартное значение |
| Целые числа         | `0`                                                          |
| Дробные числа       | `0.0`                                                        |
| `char`              | `'\0'`                                                       |
| `bool`              | `false`                                                      |
|                     |                                                              |

```csharp
default(int)            // 0
default(string)         // null
default(DateTime)       // new DateTime(1, 1, 1)

// Если контекст позволяет, скобки можно не указывать:
char symbolImplicit = default;       // '\0'
char symbolExplicit = default(char); // '\0'
bool logical = default; // false
```
