
**ASGI** (*Asynchronous Server Gateway Interface*) — это протокол взаимодействия между веб-серверами и приложениями, который является развитием более старого протокола WSGI. Основное предназначение ASGI заключается в поддержке как асинхронных, так и синхронных приложений ^0e00d8

**WSGI** (*Web Server Gateway Interface*) — это стандарт взаимодействия между веб-серверами и веб-приложениями, написанными на языке Python. Он был разработан для упрощения процесса интеграции различных веб-приложений и серверов, позволяя им работать вместе независимо от их реализации

**Marshalling** (**Маршаллинг**) — это преобразование данных между управляемыми значениями и неуправляемыми (в CLR). Можно описывать это понятие синонимами:

- Преобразование данных
- Трансформация данных
- Упаковка / распаковка данных

**Нативный (с англ. Native — родной)** — естественный, родной. Термин относится к программам, которые применяются специально для определенной операционной системы или платформы, используя ее собственные "родные" инструменты

- **Нативное приложение** — это программное обеспечение, которое устанавливается и работает непосредственно на конкретной платформе (например, iOS, Android, Windows), оптимизировано под неё, полностью использует её системные возможности и аппаратные функции устройства (камера, микрофон, геолокация и прочие) ([1](https://identityblitz.ru/glossary/native-application/)) ([2](https://ya.zerocoder.ru/pgt_wiki-nativnye-prilozheniya-opredelenie-preimushhestva-i-nedostatki/)) ([3](https://ru.wikipedia.org/wiki/%D0%9D%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D1%8B%D0%B5_%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F))
    
- Нативная разработка подразумевает использование языков и SDK, предназначенных именно для данной платформы: например, Swift или Objective-C для iOS, Kotlin или Java для Android, C# для Windows ([1](https://ya.zerocoder.ru/pgt_wiki-nativnye-prilozheniya-opredelenie-preimushhestva-i-nedostatki/)) [](https://www.simbirsoft.com/blog/nativnaya-mobilnaya-razrabotka-chto-eto-i-komu-podkhodit/).
    
- Благодаря этому нативные приложения обеспечивают **высокую производительность, стабильность, глубокую интеграцию с ОС и лучшую безопасность**, по сравнению с веб-приложениями или кроссплатформенными решениями[](https://ya.zerocoder.ru/pgt_wiki-nativnye-prilozheniya-opredelenie-preimushhestva-i-nedostatki/)[](https://www.simbirsoft.com/blog/nativnaya-mobilnaya-razrabotka-chto-eto-i-komu-podkhodit/)[](https://practicum.yandex.ru/blog/chto-takoe-nativnoe-prilozhenie/).
