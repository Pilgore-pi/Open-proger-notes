

## Структура HTTP-запроса и ответа

## Запрос

HTTP-запрос состоит из следующих частей:

1. **Стартовая строка**: Содержит метод запроса, URL и версию протокола.

```http
GET /index.html HTTP/1.1
```

2. **Заголовки**: Дополнительная информация о запросе (например, тип контента).

3. **Тело**: Данные, отправляемые на сервер (в основном для методов POST и PUT).

Пример HTTP-запроса:

```http
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

## Ответ

HTTP-ответ также имеет свою структуру:

1. **Стартовая строка**: Содержит версию протокола, статус-код и статусное сообщение.

```http
HTTP/1.1 200 OK
```

2. **Заголовки**: Дополнительная информация о ответе (например, тип контента).

3. **Тело**: Данные, возвращаемые сервером.


Пример HTTP-ответа:

```http
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>
    <body>
        <h1>Привет, мир!</h1>
    </body>
</html>
```

## Развитие протокола

С переходом к HTTP/2 произошли значительные изменения в архитектуре протокола. В отличие от текстового формата HTTP/1.1, который использует простые строки для передачи данных, HTTP/2 использует бинарный формат, что позволяет улучшить производительность за счет мультиплексирования запросов и сжатия заголовков[2](https://habr.com/ru/companies/timeweb/articles/751338/). Это означает, что клиент может отправлять несколько запросов одновременно по одному TCP-соединению без необходимости ожидания ответа на каждый из них.
