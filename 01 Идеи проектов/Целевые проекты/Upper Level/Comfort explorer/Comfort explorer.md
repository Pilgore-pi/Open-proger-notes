> Проводник, главной идеей которого является **удобство**, **комфорт использования** (в том числе приятный UI) и богатые **возможности настройки** под себя

Функционал:

- Возможность просмотра содержимого папок без перехода в папки. То есть древовидный просмотр
- **Табличный просмотр** файлов. Идеально подходит, если каталоги содержат похожие наборы файлов. Колонками таблицы могут быть: ИмяФайла, Тип (кол-во), Дата(кол-во), Размер, Каталог/НеКаталог.
    - Отображать/Не отображать сетку
- **Группировки** файлов по Типу, Размеру, Названию, Дате... Группировки могут быть вложенными
- **Фильтры**
- **Сортировка**
- **Настраиваемая стартовая страница** (последнее расположение, история дейтсвий, Избранные папки и файлы, дерево каталогов с раскрытыми первыми N уровнями (начиная от дисков))
- **Назначение псевдонимов для вкладок** проводника
- **Metadata reader**: модуль Metadata reader, позволяющий просматривать и изменять метаданные файлов.
- **Массовое переименование**; Предлагать также установить PowerToys Power rename и другие готовые программы для других ОС.
- **Массовое открытие файлов** (Несколько экземпляров приложений или, лучше, несколько вкладок в одном приложении (но это не всегда возможно))
- **Открыть как текстовый файл** (можно выбрать приложение, ассоциированное с этой операцией)
- **Открыть как бинарный файл**
- **Свойства файла**:
    - Имя, общие метаданные
    - Формат файла, определяемый из метаданных или с помощью анализа содержимого (инфа должна загружаться асинхронно; попытка определения формата может быть безуспешной)
    - Ссылка на полноценное редактирование в Metadata reader
- **Вид**:
    - Файлы
        - Таблица
        - Список (несколько колонок)
        - Список (несколько строк)
        - Плитка
    - Выравнивание (Лево, Центр, Право)
    - Размер записи (в различных вариантах представления)
    - Для всех каталогов / Для этого каталога / Для подчиненных (`[Flags] enum`)
- **Предпросмотр файла**

- **[[Yoinq — Explorer SQL-based lang|Your-SQL запросы к файлам со всем вытекающим]]**. С помощью SQL можно легко писать запросы для нахождения дубликатов файлов (соединение двух каталогов по именам файлов, размерам файлов и т.д.).
 
# Metadata editor

- Позволяет открывать свойства сразу нескольких файлов. Свойства отдельных файлов можно увидеть выбрав файл из списка. Можно редактировать метаданные и выполнять операции для всех выбранных файлов. При неуспешных попытках спрашивать "Эти файлы не удалось изменить, вы хотите отменить операцию для выбранных файлов? `Пропустить файлы`, `Отменить` `Попытаться еще раз`"

- Позволяет считывать метаданные любого файла

- Редактировать метаданные

- Выбор иконки для файла и каталога или нескольких файлов

- Определять фактический формат файла независимо от расширения, позволяет внедрять скрытое сообщение в файл.

- Позволяет открыть несколько файлов как один файл. Если открывается несколько изображений, то получается несколько слоев и сетов метаданных. Данный функционал необходим для единоразового выполнения одной операции для контента нескольких файлов. Можно открыть файлы разных форматов, тогда будет произведена попытка преобразования всех выбранных файлов к тексту, если не удается, то происходит преобразование в массив байт.

- Можно сохранять объединенный файл в формате `.union` со своими метаданными, позволяющими разделить объединение файлов обратно. М. б. просто использовать `.zip` формат

- Разделенный файл. Можно разделять файл на несколько индексируемых частей и собирать его обратно

- Шифрование и дешифрование файла с богатыми настройками

- Хеширование файла современными алгоритмами

- Компрессия и декомпрессия файла с заданными параметрами

- Найти (массив байт, текст, символ, байт, пиксель, пиксели)

- Сравнить с файлом (или объединением)

- Найти и заменить

- Привязка любого приложения для открытия файлов с заданным расширением

- По возможности, добавить возможность ассоциирования расширений файлов с сайтами, на которых будут открываться файлы. Хотя, возможно, для этого понядобятся задействование API этих сайтов. Можно просто открыть нужный сайт, без передачи туда данных, хотя бы.
