**YAML** (рекурсивный акроним: **YAML** Ain't Markup Language) — формат сериализации данных, разработанный в 2001 году, главной чертой которого является удобство человеческого восприятия.

#### Особенности

- YAML поддерживает описание основных структур данных, используемых в программировании (объекты, массивы)

- **Читаемость**: YAML ориентирован на удобочитаемость, что позволяет легко воспринимать и редактировать файлы

- **Структура данных**: В YAML данные представлены в виде пар "ключ-значение", где ключи отделяются от значений двоеточием

- **Типы данных**: YAML поддерживает различные типы данных, такие как строки, числа (целые и дробные), булевы значения и null. Также возможна явная типизация с помощью префиксов

- **Комментарии**: Комментарии начинаются с символа `#`

#### Применение YAML

YAML широко используется в различных областях, включая:

- **Конфигурационные файлы**: Один из самых распространенных способов применения YAML — создание конфигурационных файлов для приложений и сервисов. Он позволяет легко описывать параметры конфигурации и настройки, используемые при развертывании приложений

- **Инфраструктура как код (IaC)**: YAML применяется для описания конфигураций инфраструктуры, например, в Ansible для автоматизации задач или в Docker Compose для управления многоконтейнерными приложениями

- **Спецификации API**: Формат часто используется для определения спецификаций API, таких как OpenAPI, что обеспечивает краткий и понятный способ описания конечных точек и схем запросов/ответов

- **Управление контейнерами**: В DevOps YAML используется для управления контейнерами и оркестрации приложений в таких системах, как Kubernetes

- **Сериализация данных**: YAML также служит форматом для сериализации и десериализации структур данных, что упрощает обмен данными между различными системами или компонентами

#### Пример кода

Пример спецификации API в формате YAML, который описывает конечную точку для добавления или удаления очков в игре:

```yaml
openapi: 3.0.0
info:
  title: Acorn Hunt API
  version: 1.0.0
paths:
  /points:
    post:
      summary: Add or remove points from the caller in their active game
      operationId: updatePoints
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required:
                - count
              properties:
                count:
                  type: integer
                  minimum: -10
                  maximum: 10
      responses:
        '200':
          description: The points were successfully added to the player score
          content:
            application/json:
              schema:
                type: object
                required:
                  - score
                properties:
                  score:
                    type: integer
        '409':
          $ref: '#/components/responses/Conflict'
        '500':
          $ref: '#/components/responses/UnknownError'
components:
  responses:
    Conflict:
      description: Conflict occurred while processing the request
    UnknownError:
      description: An unknown error occurred
```

В этом примере спецификация описывает конечную точку `/points`, которая принимает `POST`-запрос для обновления очков игрока. Указаны требования к телу запроса и возможные ответы сервера.

Такой подход обеспечивает ясность и предсказуемость в использовании API
