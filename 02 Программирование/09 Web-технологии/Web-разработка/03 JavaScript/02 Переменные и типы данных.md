### Сравнение

**Строки:**

При сравнении строк, сравнение происходит по их кодам Unicode. Если строка состоит из нескольких символов, сравнение происходит первого с первым, второго со вторым и т. д. Как только какой-то символ больше (меньше) другого соответствующего ему символа, то результатом сравнения является результат сравнения этой пары символов.

Если строки равны до какого-то момента, но они разной длины, больше считается более длинная строка.

**Разные типы:**

```js
alert('2' > 1); // '2' неявно конвертируется в 2
alert( '01' == 1 ); // true
alert(true == 1);

let a = 0, b = "0";
alert(Boolean(a)); // false
alert(Boolean(b)); // true
alert(a == b); // true!
```

## Null

```js
// Самостоятельный тип-значение null
let age = null;

// Самостоятельный тип-значение "неопределено", значение
// по умолчанию для неинициализированных переменных
let name = undefined; // аналогично: let name;

// Операция Null-объединения
// Возвращает первое НЕНУЛЕВОЕ значение
?? (??=)

return a ?? b
//Эквивалентно
if(a == null || a == undefined) return b;
else return a;

// Null-объединение с оператором ||
// Возвращяется первое ИСТИННОЕ значение
return null || 'value' || undefined;
```

 > `null` используется для обозначения неопределенности или пустоты, `undefined` используется для получения информации об инициализированности переменной

>Оператор === сравнивает выражения без приведения.
> `null` == `undefined` (true)
> `null` === `undefined` (false)

```js
alert( null > 0 );  // false
alert( null == 0 ); // false
alert( null >= 0 ); // true
```

> Сравнение с `undefined` всегда возвращает `false`

**Null-условный оператор** возвращает `undefined`, если свойство равно `null` или `undefined`:

```js
let user = {}; // пользователь без адреса
alert(user.address); // undefined (с ошибкой)
alert(user?.address); // undefined (без ошибки)
// ИЛИ
alert(user?.[address]);

// условный вызов функции:
user.cry?.();
```

## Оператор typeof

Возвращает тип сущности в виде строки

```js
typeof 5  // "number"
typeof(5) // "number"

typeof 50 + " Квартир"; // Выведет "number Квартир"
typeof (50 + " Квартир"); // Выведет "string"

typeof Math // "object"
typeof null // "object"
typeof alert // "function"
```

`typeof null` — "object" — это ошибка в языке программирования, сохраненная для совместимости. `null` не является `object`

## Symbol

Представляет собой ID.

```js
let id1 = Symbol();
let id2 = Symbol('description');
let id3 = Symbol('description');
// id2 != id3
```

Символы в объекте-выражении:

```js
let id = Symbol("id");

let user = {
  name: "Вася",
  [id]: 123 // просто "id: 123" не сработает
};
```

> "Символы" игнорируются в цикле `for-in`
