* `git init` — создание директории с необходимой структурой с именем `.git`
* [`git add`](https://git-scm.com/docs/git-add) добавляет файлы (файл или папка) в индекс (staging area) для последующего коммита. Основные атрибуты команды:
	* `--all` — все файлы в текущем каталоге
	* `--no-all`
* [`git commit`](https://git-scm.com/docs/git-commit) — берёт все данные, добавленные в индекс с помощью `git add`, и сохраняет их слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок
	* `-a | --all` — автоматическое помещение изменных и удаленных данных в Stage area
	* `-p | --patch` — Use the interactive patch selection interface to choose which changes to commit
	* `-m "<msg>" | --message "<msg>"` — устанавливает сообщение для коммита, при указании нескольких атрибутов `--m`, их сообщения будут совмещены как отдельные параграфы.
* `git status` — показывает состояния файлов в рабочем каталоге и индексе: какие файлы изменены, но не добавлены в индекс; какие ожидают коммита в индексе
* `git log` — показывает логи коммитов. Логи содержат идентификаторы коммитов, на которые можно переключиться (откатиться) с помощью команды `git checkout <commit_id>`

Команды синхронизации с удаленным сервером [(1)](https://selectel.ru/blog/tutorials/what-is-git-push-and-how-to-use-it/):

* `git push` — позволяет отправлять локальную ветку на удаленный репозиторий. Если программист работает один, то пуш позволяет хранить код в облаке, например github, gitlab
* `git pull` — получение изменений с сервера
* `git remote` — получение списка удаленных подключений к репозиторию

Ветви:

>Имя главной ветки — `master`. Главная ветка только логически отличается от других веток, физически она не отличается

* `git branch branch_name` — создание ветви `branch_name`
	* `-d | --delete` — удалить ветку
* `git checkout b_name` — переключение на ветку
	* `-b` — создать и переключиться на созданную ветку
* `git merge b_name` — сливает ветку `b_name` в текущей веткой (выбранную с помощью команды `git checkout`)
* `git diff` — используется для вычисления разницы между любыми двумя Git деревьями
